# <a name="get-started-with-nodejs-web-frameworks-on-windows"></a>Приступая к работе с веб-платформами Node. js в Windows

Пошаговое руководство, помогающее приступить к работе с веб-платформами Node. js в Windows, включая Next. js, Нукст. js и Гатсби.

## <a name="prerequisites"></a>Предварительные требования

В этом руководстве предполагается, что вы уже выполнили действия по [настройке среды разработки Node. js с помощью WSL 2](./setup-on-wsl2.md), включая:

- Установите Windows 10 Insider Preview Build 18932 или более поздней версии.
- Включите функцию WSL 2 в Windows.
- Установите дистрибутив Linux (Ubuntu 18,04 для наших примеров). Это можно проверить с помощью: `wsl lsb_release -a`
- Убедитесь, что дистрибутив Ubuntu 18,04 работает в режиме WSL 2. (WSL может запускать дистрибутивы в режиме v1 или v2.) Это можно проверить, открыв PowerShell и введя: `wsl -l -v`.
- С помощью PowerShell в качестве дистрибутива по умолчанию устанавливается Ubuntu 18,04 с: `wsl -s ubuntu 18.04`.

## <a name="get-started-with-nextjs"></a>Начало работы с Next. js

Next. js — это платформа для создания отрисованных сервером приложений JavaScript на основе реагируют. js, Node. js, веб-пакета и Babel. js. По сути, это шаблон проекта, предназначенный для реагирования, в котором особое внимание уделяется рекомендациям, который позволяет создавать «универсальные» веб-приложения простым и согласованным способом с недействительной конфигурацией. Эти «универсальные» веб-приложения, отображаемые на сервере, иногда называют «сходны», что означает, что код является общим для клиента и сервера.

Чтобы создать следующий проект. js, который включает установку следующего, реагирования и реагирования DOM:

1. Откройте терминал WSL (Internet Explorer). Ubuntu 18,04).

2. Создайте новую папку проекта: `mkdir NextProjects` и введите этот каталог: `cd NextProjects`.

3. Установите Next. js и создайте проект (заменив "My-Next-App" на то, что вы хотите вызвать приложение): `npm create next-app my-next-app`.

4. После установки пакета измените каталоги в новой папке приложения, `cd my-next-app`, а затем используйте `code .`, чтобы открыть проект Next. js в VS Code. Это позволит просмотреть следующую платформу. js, созданную для приложения. Обратите внимание, что VS Code открыло приложение в среде WSL-Remote (как указано на зеленой вкладке в левом нижнем углу окна VS Code). Это означает, что хотя вы используете VS Code для редактирования в ОС Windows, приложение по-прежнему работает в ОС Linux.

    ![WSL — удаленное расширение](../images/wsl-remote-extension.png)

5. Есть три команды, которые необходимо знать после установки Next. js:

    - `npm run dev` для запуска экземпляра разработки с возможностью горячей перезагрузки, просмотра файлов и повторного выполнения задач.
    - `npm run build` для компиляции проекта.
    - `npm start` для запуска приложения в рабочем режиме.

    Откройте терминальный WSL, интегрированный в VS Code (**просмотр > терминал**). Убедитесь, что путь терминала указывает на каталог проекта (IE. `~/NextProjects/my-next-app$`). Затем попробуйте запустить экземпляр разработки нового приложения следующего JS с помощью: `npm run dev`.

6. Локальный сервер разработки будет запущен, после того как страницы проекта будут завершены, в окне терминала будет отображаться "скомпилировано успешно" в [http://localhost:3000](http://localhost:3000)". Выберите эту ссылку localhost, чтобы открыть новое приложение Next. js в веб-браузере.

    ![Следующее приложение. js, выполняемое на localhost: 3000](../images/next-app.png)

7. Откройте файл `pages/index.js` в редакторе VS Code. Найдите заголовок страницы `<h1 className='title'>Welcome to Next.js!</h1>` и измените его на `<h1 className='title'>This is my new Next.js app!</h1>`. Если веб-браузер по-прежнему открыт для localhost: 3000, сохраните изменения и обратите внимание, что функция горячей перезагрузки автоматически компилирует и обновляет изменения в браузере.

8. Давайте посмотрим, как Next. js обрабатывает ошибки. Удалите закрывающий тег `</h1>`, чтобы код заголовка теперь выглядит следующим образом: `<h1 className='title'>This is my new Next.js app!`. Сохраните это изменение и обратите внимание, что в браузере отображается ошибка "ошибка компиляции", и в терминале вы узнаете, что закрывающий тег для `<h1>`. Замените закрывающий тег `</h1>`, сохраните, и страница будет перезагружена.

Отладчик VS Code можно использовать со следующим приложением. js, нажав клавишу F5 или **просмотрев > Отладка** (Ctrl + Shift + D) и **Просмотреть > консоль отладки** (Ctrl + Shift + Y) в строке меню. Если выбрать значок шестеренки в окне Отладка, будет создан файл конфигурации запуска (`launch.json`) для сохранения сведений о настройке отладки. Дополнительные сведения см. в разделе [VS Codeная Отладка](https://code.visualstudio.com/docs/editor/debugging#_launch-configurations).

![Значок настройки окна отладки VS Code и launch. JSON](../images/vscode-debug-launch-configuration.png)

Дополнительные сведения о следующем JS см. в документации по [следующему JS](https://nextjs.org/docs).

## <a name="get-started-with-nuxtjs"></a>Начало работы с Нукст. js

Нукст. js — это платформа для создания отрисованных сервером приложений JavaScript на основе Vue. js, Node. js, веб-пакета и Babel. js. Это было повышено в следующем JS. По сути, это стандартный проект для Vue. Как и в случае со следующим JS-приложением, он создает особое внимание рекомендациям и позволяет создавать «универсальные» веб приложения простым и согласованным способом, с невысокой настройкой. Эти «универсальные» веб-приложения, отображаемые на сервере, иногда называют «сходны», что означает, что код является общим для клиента и сервера.

Чтобы создать проект Нукст. js, который будет включать ответы на ряд вопросов о том, что такое Интегрированная серверная платформа, платформа пользовательского интерфейса, платформа тестирования, режим, модули и средство поиска ошибок, которые вы хотите установить:

1. Откройте терминал WSL (Internet Explorer). Ubuntu 18,04).

2. Создайте новую папку проекта: `mkdir NuxtProjects` и введите этот каталог: `cd NuxtProjects`.

3. Установите Нукст. js и создайте проект (заменив "My-нукст-App" на то, что вы хотите вызвать приложение): `npm create nuxt-app my-nuxt-app`.

4. Установщик Нукст. js теперь будет задавать следующие вопросы:
    - Имя проекта: My-нукстжс-App
    - Описание проекта: Описание приложения Нукст. js.
    - Имя автора: Я использую свой псевдоним GitHub.
    - Выберите Диспетчер пакетов: Yarn или **NPM** . Мы используем NPM в нашем примере.
    - Выберите платформу пользовательского интерфейса: Нет, Ant Design Vue, начальной загрузки, Vue и т. д. Давайте выберем **вуетифи** в этом примере, но сообщество Vue создало удобную [сводку по сравнению этих платформ пользовательского интерфейса](https://vue-community.org/guide/ecosystem/ui-libraries.html#summary-tldr) , чтобы помочь вам выбрать оптимальное соответствие для проекта.
    - Выберите пользовательские серверные платформы: None, Адонисжс, Express, Фастифи и т. д. Давайте выберем « **нет** » в этом примере, но на сайте dev.to можно найти [Сравнение платформы 2019-2020 Server](https://dev.to/santypk4/introducing-the-best-10-node-js-frameworks-for-2019-and-2020-mcm) .
    - Выберите модули Нукст. js (используйте пробел, чтобы выбрать модули или просто введите, если не хотите): Аксиос (для упрощения HTTP-запросов) или [Поддержка PWA](https://pwa.nuxtjs.org/) (для добавления файла службы-исполнителя, manifest. JSON и т. д.). Не добавим модуль для этого примера.
    - Выберите linting Tools: **ESLint**, аккуратнее, Lint промежуточные файлы. Давайте выберем **ESLint** (инструмент для анализа кода и предупреждения о возможных ошибках).
    - Выберите платформу тестирования: **None**, ЖЕСТ, Ava. Давайте выберем « **нет** », так как мы не будем охватывать тестирование в этом кратком руководстве.
    - Выберите режим рендеринга: **Универсальное (SSR)** или одностраничное приложение (SPA). Давайте выбираем в нашем примере **универсальные (SSR)** , но в [документах нукст. js](https://nuxtjs.org/guide#server-rendered-universal-ssr-) указываются некоторые различия — SSR, которым требуется сервер Node. js, работающий на сервере, визуализируйте приложение и SPA для статического размещения.
    - Выберите средства разработки: **jsconfig. JSON** (рекомендуется для VS Code, чтобы функция завершения кода IntelliSense работала)

5. После создания проекта `cd my-nuxtjs-app`, чтобы войти в каталог проекта Нукст. js, а затем введите `code .`, чтобы открыть проект в VS Code WSL-Remote Environment.

    ![WSL — удаленное расширение](../images/wsl-remote-extension.png)

6. Существует три команды, которые необходимо знать после установки Нукст. js:

    - `npm run dev` для запуска экземпляра разработки с возможностью горячей перезагрузки, просмотра файлов и повторного выполнения задач.
    - `npm run build` для компиляции проекта.
    - `npm start` для запуска приложения в рабочем режиме.

    Откройте терминальный WSL, интегрированный в VS Code (**просмотр > терминал**). Убедитесь, что путь терминала указывает на каталог проекта (IE. `~/NuxtProjects/my-nuxt-app$`). Затем попробуйте запустить экземпляр разработки нового приложения Нукст. js, используя: `npm run dev`

6. Будет запущен локальный сервер разработки (на котором будут отображаться некоторые типы интересных индикаторов выполнения для клиента и сервера). После завершения создания проекта в терминале будет отображаться «компиляция выполнена успешно» вместе с количеством времени, затраченного на компиляцию. Укажите для веб-браузера значение [http://localhost:3000](http://localhost:3000) , чтобы открыть новое приложение нукст. js.

    ![Приложение Нукст. js, работающее на локальном узле localhost: 3000](../images/nuxt-app.png)

7. Откройте файл `pages/index.vue` в редакторе VS Code. Найдите заголовок страницы `<v-card-title class="headline">Welcome to the Vuetify + Nuxt.js template</v-card-title>` и измените его на `<v-card-title class="headline">This is my new Nuxt.js app!</v-card-title>`. Если веб-браузер по-прежнему открыт для localhost: 3000, сохраните изменения и обратите внимание, что функция горячей перезагрузки автоматически компилирует и обновляет изменения в браузере.

8. Давайте посмотрим, как Нукст. js обрабатывает ошибки. Удалите закрывающий тег `</v-card-title>`, чтобы код заголовка теперь выглядит следующим образом: `<v-card-title class="headline">This is my new Nuxt.js app!`. Сохраните это изменение и обратите внимание, что в браузере будет отображаться ошибка компиляции, и в окне терминала появится сообщение о том, что отсутствует закрывающий тег для `<v-card-title>`, а также номера строк, где ошибка может быть найдена в коде. Замените закрывающий тег `</v-card-title>`, сохраните, и страница будет перезагружена.

Можно использовать отладчик VS Code в приложении Нукст. js, нажав клавишу F5 или **просмотрев > Отладка** (Ctrl + Shift + D) и **Просмотреть > консоль отладки** (Ctrl + Shift + Y) в строке меню. Если выбрать значок шестеренки в окне Отладка, будет создан файл конфигурации запуска (`launch.json`) для сохранения сведений о настройке отладки. Дополнительные сведения см. в разделе [VS Codeная Отладка](https://code.visualstudio.com/docs/editor/debugging#_launch-configurations).

## <a name="get-started-with-gatsbyjs"></a>Начало работы с Гатсби. js

Гатсби. js — это статическая платформа генератора сайтов, основанная на реагируют. js, в отличие от отрисовки на сервере, такой как Next. js и Нукст. js. Генератор статических сайтов создает статический HTML во время сборки. Сервер не требуется. Next. js и Нукст. js создают HTML в среде выполнения (каждый раз, когда поступает новый запрос). Для их запуска требуется сервер. Гатсби также определяет обработку данных в приложении (с помощью Графкл), тогда как следующие. js и Нукст. js оставляют это решение.

Чтобы создать проект Гатсби. js, выполните следующие действия.

1. Откройте терминал WSL (Internet Explorer). Ubuntu 18,04).
2. Создайте новую папку проекта: `mkdir GatsbyProjects` и введите этот каталог: `cd GatsbyProjects`
3. Используйте NPM для установки Гатсби CLI: `npm install -g gatsby-cli`. После установки проверьте версию с `gatsby --version`.
4. Создайте проект Гатсби. js: `gatsby new my-gatsby-app`
5. После установки пакета измените каталоги в новой папке приложения, `cd my-gatsby-app`, а затем используйте `code .`, чтобы открыть проект Гатсби в VS Code. Это позволит просмотреть платформу Гатсби. js, созданную для приложения, с помощью проводника VS Code. Обратите внимание, что VS Code открыло приложение в среде WSL-Remote (как указано на зеленой вкладке в левом нижнем углу окна VS Code). Это означает, что хотя вы используете VS Code для редактирования в ОС Windows, приложение по-прежнему работает в ОС Linux.

    ![WSL — удаленное расширение](../images/wsl-remote-extension.png)

6. Есть три команды, которые необходимо знать после установки Гатсби:

    - `gatsby develop` для запуска экземпляра разработки с горячей перегрузкой.
    - `gatsby build` для создания рабочей сборки.
    - `gatsby serve` для запуска приложения в рабочем режиме.

    Откройте терминальный WSL, интегрированный в VS Code (**просмотр > терминал**). Убедитесь, что путь терминала указывает на каталог проекта (IE. `~/GatsbyProjects/my-gatsby-app$`). Затем попробуйте запустить экземпляр разработки нового приложения, используя: `gatsby develop`

7. После завершения компиляции нового проекта Гатсби в окне терминала отобразится: "теперь вы можете просмотреть Гатсби-Starter-Default в браузере. [http://localhost:8000/](http://localhost:8000/)». Щелкните эту ссылку localhost, чтобы просмотреть новый проект, созданный в веб-браузере.

> [!NOTE]
> Вы заметите, что в выходных данных терминала есть возможность "Просмотреть Графикл, интегрированную среду разработки в браузере, чтобы исследовать данные и схему вашего сайта: [http://localhost:8000/___graphql](http://localhost:8000/___graphql)". Графкл объединяет ваши интерфейсы API в самостоятельный документ IDE (Графикл), встроенный в Гатсби. Помимо изучения данных и схемы сайта, можно выполнять такие операции Графкл, как запросы, изменения и подписки. Дополнительные сведения см. в разделе [Введение в графикл](https://www.gatsbyjs.org/docs/running-queries-with-graphiql/).

8. Откройте файл `src/pages/index.js` в редакторе VS Code. Найдите заголовок страницы `<h1 >Hi people</h1>` и измените его на `<h1 >Hi (Your Name)!</h1>`. Если веб-браузер по-прежнему открыт для localhost: 8000, сохраните изменения и обратите внимание, что функция горячей перезагрузки автоматически компилирует и обновляет изменения в браузере.

    ![Приложение Гатсби. js, работающее на локальном узле localhost: 3000](../images/gatsby-app.png)

9. Давайте посмотрим, как Next. js обрабатывает ошибки. Удалите закрывающий тег `</h1>`, чтобы код заголовка теперь выглядит следующим образом: `<h1>Hi (Your Name)!`. Сохраните это изменение и обратите внимание, что в браузере отображается ошибка "ошибка компиляции", и в терминале вы узнаете, что закрывающий тег для `<h1>`. Замените закрывающий тег `</h1>`, сохраните, и страница будет перезагружена.

Отладчик VS Code можно использовать со следующим приложением. js, нажав клавишу F5 или **просмотрев > Отладка** (Ctrl + Shift + D) и **Просмотреть > консоль отладки** (Ctrl + Shift + Y) в строке меню. Если выбрать значок шестеренки в окне Отладка, будет создан файл конфигурации запуска (`launch.json`) для сохранения сведений о настройке отладки. Дополнительные сведения см. в разделе [VS Codeная Отладка](https://code.visualstudio.com/docs/editor/debugging#_launch-configurations).
