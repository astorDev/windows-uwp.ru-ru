# <a name="choose-your-app-platform"></a>Выберите платформу приложения

Вы хотите создать новый классическое приложение для компьютеров под управлением Windows, первое решение, внесенные при платформу, на которой приложение для использования. Windows предоставляет четыре платформы основного приложения, каждый из которых свои сильные стороны:

* [Универсальная платформа Windows (UWP)](#uwp)
* [WPF (.NET)](#wpf)
* [Windows Forms (.NET)](#windows-forms)
* [Win32](#win32)

Все эти платформы приложения позволяют создавать Классические приложения, такими как Word, Excel и Photoshop, которые работают в классическом Windows desktop и take всех преимуществ от определенных функций этой среды. Тем не менее некоторые из этих платформ совместно используют некоторые признаки и лучше подходят для определенных типов приложений:

* **UWP, WPF и Windows Forms**. Эти платформы предоставляют управляемые среды исполнения (среду выполнения Windows, для которого для универсальной платформы Windows и .NET Windows Forms и WPF) много преимуществ, особенно в области производительности труда разработчиков, сложные и настраиваемый пользовательский Интерфейс и безопасность приложений. Так как эти платформы поддерживают визуальные конструкторы и разметки пользовательского интерфейса для быстрого создания пользовательского интерфейса, они особенно хорошо подходит для бизнес-приложений.

* **Win32 API**. Win32 API (также называемый Windows API) — это исходный платформа машинном языке C /C++ приложения Windows, которым требуется прямой доступ к Windows и оборудования. Предоставляет возможности разработки первого класса без зависимости от того, управляемая среда, таких как .NET и WinRT. Это делает предпочтительной платформой для приложений, которым требуется высокий уровень производительности и прямой доступ к оборудованию системы Win32 API.

В этой статье описываются эти платформы более подробно и поможет вам определить наиболее подходящий вариант для вашего приложения.

## <a name="uwp"></a>UWP

UWP — непревзойденной платформа для игр и приложений Windows 10. Это чрезвычайно гибкие платформы, которая использует разметки XAML для отделения пользовательского интерфейса (представления) от кода (бизнес-логики). UWP подходит для классических приложений, что, требуют усложненный пользовательский Интерфейс, настройки стилей и сценарии графикой. UWP также имеет встроенную поддержку [Fluent Design System](/windows/uwp/design/fluent-design-system/) по умолчанию UX столкнуться и предоставляет доступ к [среды выполнения Windows (WinRT) API-интерфейсы](/windows/uwp/get-started/universal-application-platform-guide#how-the-universal-windows-platform-relates-to-windows-runtime-apis). Используя Fluent, универсальной платформы Windows автоматически поддерживает распространенные методы ввода, например рукописный ввод, сенсорного ввода, игровой, клавиатуры и мыши.

Не только служит универсальной платформы Windows для создания настольных приложений для компьютеров под управлением Windows, но универсальной платформы Windows также поддерживается только платформа для приложений Xbox, HoloLens и Surface Hub. UWP является нашей платформы по самому новому, передовых приложений.

Дополнительные сведения о UWP см. в разделе [приступить к работе с приложениями Windows 10](/windows/uwp/get-started/).

## <a name="wpf"></a>WPF

WPF является установленной платформы для управляемых приложений Windows с доступом к полной версии платформы .NET Framework и разметки XAML также используется для разделения UX из кода. Эта платформа предназначена для настольных приложений, требующих усложненный пользовательский Интерфейс, настройки стилей и сценарии графикой. Навыки разработки WPF аналогичны навыков разработки универсальной платформы Windows, поэтому миграции из WPF в приложениях универсальной платформы Windows проще, чем переход с Windows Forms.

Дополнительные сведения о WPF, см. в разделе [Приступая к работе (WPF)](https://docs.microsoft.com/dotnet/framework/wpf/getting-started/).

## <a name="windows-forms"></a>Windows Forms

Windows Forms является исходной платформой для управляемых приложений Windows с помощью упрощенного модель пользовательского интерфейса и доступ к полной версии платформы .NET Framework. Он является превосходным средством позволяет разработчикам быстро приступить к созданию приложений, даже для разработчиков — новичков в платформе. Это — это платформа разработки приложений на основе форм, быстро с помощью встроенных большое количество визуальные и невизуальные элементы управления и перетащите. Windows Forms использует XAML, поэтому принятие решения об позже расширить ваше приложение UWP влечет за собой полный перезаписи пользовательского интерфейса.

Дополнительные сведения о Windows Forms, см. в разделе [Приступая к работе с Windows Forms](https://docs.microsoft.com/dotnet/framework/winforms/getting-started-with-windows-forms).

## <a name="win32"></a>Win32

С помощью API Win32 с C++ дает возможность обеспечить наивысшую производительность и эффективность, используя более строгий контроль целевой платформы с неуправляемым кодом, чем возможно в управляемой среде выполнения как WinRT и .NET. Тем не менее применение такой уровень контроля над выполнением вашего приложения требуется больше и сопровождения, иначе трудной и меняет эффективности разработки производительность во время выполнения.

Ниже приведены преимущества Win32 API и C++ предлагает позволяют создавать высокопроизводительные приложения.

-   Оптимизация аппаратного уровня, включая жесткий контроль над выделения ресурсов, время жизни объектов, структура данных, выравнивание, байтов упаковки и многое другое.
-   Доступ к ориентирована на производительность инструкций наборами как расширения SSE и AVX через встроенные функции.
-   Эффективный, типобезопасный обобщенное программирование с помощью шаблонов.
-   Алгоритмы и контейнеры эффективный и безопасный.
-   DirectX в определенной Direct3D и DirectCompute (Обратите внимание, что универсальной платформы Windows также предлагает взаимодействия DirectX).
-   C++AMP.

Дополнительные сведения см. в разделе [приступить к работе с классических приложений Windows, которые используют Win32 API](/windows/desktop/desktop-programming) и [технологий настольных приложений](/windows/desktop/desktop-app-technologies).

### <a name="win32-and-c-for-traditional-desktop-applications"></a>Win32 и C++ традиционных приложений для настольных компьютеров

При записи в приложении для настольных систем C++, вы можете выбрать Win32 или MFC для пользовательского интерфейса, или узел платформ сторонних приложений, поддерживающих платформ, отличных от Windows.

-   **Win32:** Это дескриптор API на основе языка C платформы Windows, включая элементы управления, но не ограничиваясь ими функциональные возможности пользовательского интерфейса, такие как управление окнами, рисования и пользовательского интерфейса. Поскольку низкого уровня, язык C API, основанный на маркеры, это редко Выбор для создания современных, интенсивно пользовательского интерфейса приложения. Тем не менее он предоставляет базовые интерфейсы API, необходимые для взаимодействия с платформой Windows и является подходящим выбором для приложений имеют простой пользовательский Интерфейс требования или, вам требуется только Windows пользовательского интерфейса, чтобы оставаться в сторону максимально, к примеру, игры.
-   **MFC (библиотеки Microsoft Foundation Class):** Это платформа почтенный приложений и библиотека пользовательского интерфейса, который обслуживает разработчиков Windows с 1992 года. Это тонкий C++ программа-оболочка дескриптора на основе, язык C API Win32 и предоставляет объектно ориентированного интерфейса для многих стандартных windows, стандартные элементы управления и другие объекты Windows. Несмотря на то, что многие современные платформы пользовательского интерфейса в экосистеме .NET преодолеть MFC в удобства, он по-прежнему собственного пользовательского интерфейса прикладной платформы для многих C++ разработчиков, создающих приложения для настольных систем Windows.
-   **Платформ сторонних приложений:** Так как C++ может выполняться на различных платформах и не привязана к Windows или среды выполнения .NET, третьим сторонам разработали нового приложения и платформы пользовательского интерфейса для C++ для облегчения разработки кросс платформенных приложений с помощью многофункциональных пользовательских интерфейсов. Некоторые из этих платформ предоставляют свои собственные интерфейса и вид, другим пользователям, например wxWidgets или Qt использовать или эмуляцию собственный элемент управления набор платформы. Использование этих библиотек, его можно совместно использовать практически все приложения исходного кода между версиями приложения под управлением Windows или других платформ, таких как OSX или Linux.

Используйте Xamarin для создания кроссплатформенных приложений для Windows 10, который также можно запустить на iOS и Android. Дополнительные сведения см. в разделе [Xamarin](https://docs.microsoft.com/xamarin/xamarin-forms/get-started/index).
