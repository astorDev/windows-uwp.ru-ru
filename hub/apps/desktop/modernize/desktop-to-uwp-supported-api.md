# <a name="uwp-apis-available-to-desktop-apps"></a>Интерфейсы API UWP, доступные для классических приложений

Классические приложения могут использовать широкий спектр API универсальная платформа Windows (UWP), даже если они не были полностью перенесены в приложение UWP. В этой статье перечислены доступные классы, которые может использовать классическое приложение.

Для многих API-интерфейсов UWP [удостоверение пакета](modernize-packaged-apps.md) необходимо использовать в классическом приложении. Большинство интерфейсов API UWP работают с хорошо работающими приложениями, которые имеют удостоверение пакета. Однако некоторые функциональные области пока еще не прошли полное тестирование или работают неправильно.

| Область применения компонента | Подробности |
|--------------|---------|
| Встречи, контакты и сообщения электронной почты | В настоящее время поддерживаются не все API. |
| Проверка подлинности и удостоверение пользователя | В настоящее время поддерживаются не все API. |
| Фоновые задачи | Пока классические приложения могут регистрировать фоновые задачи, только полностью приложения UWP могут выполняться <i>как</i> фоновая задача. Дополнительные сведения см. в разделе [Пример фоновой задачи на GitHub](https://github.com/Microsoft/DesktopBridgeToUWP-Samples/tree/master/Samples/BackgroundTasksSample). |
| "Bluetooth" | В настоящее время поддерживаются не все API. |
| Чат и телефонные звонки | Классические приложения не могут работать на мобильных устройствах, пока они не будут полностью перенесены в UWP. |
| Активация контракта | Только полные приложения UWP могут быть активированы контрактом. |
| Cryptography | В настоящее время поддерживаются не все API. |
| Геолокация | В настоящее время поддерживаются не все API. |
| Средства выбора файлов и папок | Приложения с удостоверением имеют полный доступ к файловой системе и не требуют выбора UWP. |
| Media | В настоящее время поддерживаются не все API. |
| Точка обслуживания | В настоящее время поддерживаются не все API. |
| Смарт-карты | В настоящее время поддерживаются не все API. |
| PlayReady | В настоящее время поддерживаются не все API. |
| "VPN" | В настоящее время поддерживаются не все API. |
| Кошелек | Классические приложения не могут работать на мобильных устройствах, пока они не будут полностью перенесены в UWP. |
| Пользовательский интерфейс XAML | Windows 10, версия 1809 и более ранние версии не поддерживают размещение содержимого XAML UWP в классических приложениях. Начиная с Windows 10 версии 1903, вы можете использовать [острова XAML](xaml-islands.md) для размещения содержимого XAML UWP.  |

Иногда API в этих областях могут работать правильно. Однако если API явно не указан в списке поддерживаемых, следует избегать его использования, так как возможны непредусмотренные побочные эффекты или поведение. Поддержка этих API может появиться в будущих выпусках. Пока же следует использовать в вашем приложении соответствующие API Win32 или .NET.

### <a name="apis-supported-in-all-desktop-applications"></a>API-интерфейсы, поддерживаемые во всех настольных приложениях

API-интерфейсы с [дуалапипартитион](https://docs.microsoft.com/uwp/api/Windows.Foundation.Metadata.DualApiPartitionAttribute) поддерживаются во всех классических приложениях, включая те, которые имеют удостоверение пакета и без удостоверения пакета. Чтобы определить, поддерживается ли API, который требуется вызвать из приложения для настольных систем, найдите справочную статью по интересующему вас классу или участнику в [справочной документации по пространствам имен WinRT](https://docs.microsoft.com/uwp/api/). Затем проверьте, содержит ли раздел **Attributes** в справочной статье атрибут [дуалапипартитион](https://docs.microsoft.com/uwp/api/Windows.Foundation.Metadata.DualApiPartitionAttribute) .
