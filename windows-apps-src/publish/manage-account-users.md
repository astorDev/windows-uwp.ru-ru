# <a name="manage-account-users"></a>Управление пользователями учетных записей


Для добавления пользователей в свою учетную запись Центра разработки можно использовать Azure Active Directory. Каждому пользователю назначается роль, которая дает ему определенный набор разрешений на учетную запись. Вы также можете назначить роль группе пользователей или приложению Azure Active Directory.

> **Важно!** Чтобы добавлять пользователей учетной записи и управлять ими, сначала нужно связать учетную запись Центра разработки с Azure Active Directory вашей организации. Для этого необходимо войти в Azure Active Directory, используя учетную запись [глобального администратора](http://go.microsoft.com/fwlink/?LinkId=746654). Установив эту связь, вы не сможете ее удалить без помощи службы поддержки.

## <a name="associate-your-dev-center-account-with-your-organizations-azure-active-directory"></a>Как связать учетную запись Центра разработки с Azure Active Directory организации

Центр разработки для Windows использует Azure Active Directory для управления пользователями и назначения им ролей. Если ваша организация уже использует Office 365 или другие бизнес-службы от Майкрософт, то у вас уже есть Azure Active Directory. В противном случае можно создать новую Azure AD из Центра разработки без дополнительной платы.

Учтите, что с Azure AD можно связать только одну учетную запись Центра разработки. Аналогичным образом, только один каталог Azure AD можно связать с учетной записью Центра разработки.

> **Примечание**. Если пользователи, которых вы хотите добавить, не входят в Azure AD вашей организации и вы не хотите создавать для них новые учетные записи Azure AD, можно [пригласить пользователей по электронной почте](#add-and-manage-account-users).

### <a name="associate-your-dev-center-account-with-your-organizations-existing-azure-ad"></a>Связывание учетной записи Центра разработки с существующей Azure AD организации

Если ваша организация уже использует Azure AD, выполните следующие действия, чтобы связать свою учетную запись Центра разработки.

1.  Перейдите на страницу **Настройка учетной записи** и выберите пункт **Управление пользователями**.
2.  Нажмите кнопку **Связать Azure AD со своей учетной записью Центра разработки**.
3.  Войдите в свою учетную запись Active Directory. Чтобы установить связь, у учетной записи должны быть разрешения [глобального администратора](http://go.microsoft.com/fwlink/?LinkId=746654).
4.  Проверьте название организации и доменное имя для учетной записи Azure Active Directory. Чтобы установить связь, нажмите кнопку **Подтвердить**.
5.  Если связь успешно установлена, вы сможете добавлять пользователей учетной записи и управлять ими на странице **Управление пользователями**, как описано в разделах ниже.

### <a name="create-a-brand-new-azure-ad-to-associate-with-your-dev-center-account"></a>Создание новой службы Azure AD для связывания со своей учетной записью Центра разработки

Если необходимо настроить новую службу Azure AD, чтобы связать ее со своей учетной записью Центра разработки, выполните следующие действия.

1.    Перейдите на страницу **Настройка учетной записи** и выберите пункт **Управление пользователями**.
2.    Нажмите кнопку **Создать новую службу Azure AD**.
3.    Введите сведения для внесения новой службы Azure AD в каталог:
 - **Доменное имя**: уникальное имя, которое будет использоваться для вашего домена Azure AD вместе с «.onmicrosoft.com». Например, если вы укажете здесь example, домен Azure AD будет иметь вид example.onmicrosoft.com.
 - **Контактный адрес электронной почты**: адрес электронной почты, по которому мы сможем при необходимости связаться с вами по вопросам, связанным с вашей учетной записью.
 - **Сведения о пользовательской учетной записи глобального администратора**: имя, фамилия, имя пользователя и пароль для новой учетной записи администратора.
4.    Щелкните **Создать**, чтобы подтвердить новые сведения о домене и учетной записи.
5.    Выполните вход с именем пользователя и паролем глобального администратора Azure AD, чтобы начать добавлять дополнительных пользователей учетных записей и управлять ими на странице **Управление пользователями** вашей учетной записи, как описано ниже.


> **Важно!** После связывания своей учетной записи в Центре разработки с Azure AD для добавления пользователей и управления ими потребуется выполнять вход в Центр разработки, только используя учетную запись глобального администратора Azure AD (а не личную учетную запись Майкрософт).

## <a name="add-and-manage-account-users-groups-and-azure-ad-applications"></a>Добавление пользователей, групп и приложений Active Directory в учетную запись и управление ими

Установив связь, вы сможете добавлять в свою учетную запись пользователей, группы и приложения Azure Active Directory. Кроме того, вы сможете менять роли, редактировать сведения об учетной записи и удалять пользователей.

> **Примечание.** Если ваша организация использует [интеграцию каталогов](http://go.microsoft.com/fwlink/p/?LinkID=724033) для синхронизации локальной службы каталогов с Azure AD, вы не сможете создавать новых пользователей, группы и приложения Azure AD в Центре разработки. Вам (или другому администратору локального каталога) придется создавать их непосредственно в локальном каталоге, чтобы их можно было добавить в Центр разработки.

При управлении пользователями учитывайте следующее:

-   Все пользователи Центра разработки должны иметь активную учетную запись в Azure Active Directory вашей организации.
-   При создании **нового** пользователя или группы в Центре разработки они будут добавлены и в Azure Active Directory вашей организации.
-   При изменении имени пользователя или группы в Центре разработки эти изменения будут отражены и в Azure Active Directory вашей организации.
-   Пользователи (в том числе группы и приложения Azure Active Directory) получат доступ ко всей учетной записи Центра разработчиков в соответствии с разрешениями, которые связаны с назначенной им ролью. Если вы [настраиваете разрешения](set-custom-permissions-for-account-users.md), вы можете ограничить доступ пользователя, чтобы он мог работать только с определенными приложениями и (или) надстройками.
-   Вы можете разрешить пользователю, группе или приложению Azure AD получать доступ к функциональности нескольких ролей, выбрав несколько ролей, или с помощью [пользовательских разрешений](set-custom-permissions-for-account-users.md) для предоставления требуемых прав доступа.
-   Пользователь с определенной ролью (или набором [пользовательских разрешений](set-custom-permissions-for-account-users.md)) также может быть членом группы с другой ролью (или набором разрешений). В этом случае пользователь будет иметь доступ ко всем возможностям, связанным и с отдельной учетной записью, и с группой.

## <a name="roles-and-permissions"></a>Роли и разрешения

При добавлении пользователя, группы или приложения Azure AD необходимо указать их разрешения. Это можно сделать, назначая им **стандартную роль** или [настраивая их разрешения](set-custom-permissions-for-account-users.md).

Если вы не используете пользовательские разрешения, всем пользователям, группам или приложениям Azure Active Directory, добавляемым в учетную запись, необходимо назначить по крайней мере одну из указанных ниже стандартных ролей. Каждая роль имеет определенный набор разрешений для выполнения определенных функций в учетной записи. 

> **Примечание.** Владельцем учетной записи является пользователь, создавший ее с помощью учетной записи Майкрософт (а не пользователи, добавленные с помощью Azure Active Directory). Только владелец имеет полный доступ к учетной записи: он может удалять приложения, создавать и редактировать любых пользователей, а также изменять финансовую информацию и параметры учетной записи. 

| Роль                 | Описание              |
|----------------------|--------------------------|
| Менеджер              | Имеет полный доступ к учетной записи, за исключением права на изменение параметров налогов и выплат. Сюда относится и управление пользователями в Центре разработки, но обратите внимание, что право на создание и удаление пользователей зависит от разрешений в Azure Active Directory. То есть, если пользователю назначена роль менеджера, но в Azure Active Directory у него нет разрешений администратора, он не сможет создавать или удалять пользователей из каталога (но сможет изменять их роли в Центре разработки). |
| Разработчик            | Может отправлять пакеты, приложения и надстройки, а также просматривать сведения телеметрии в [отчете об использовании](usage-report.md). Не может просматривать финансовую информацию и параметры учетной записи.   |
| Бизнес-участник | Может просматривать отчеты по [Работоспособности](health-report.md) и [Использованию](usage-report.md). Не может создавать и отправлять продукты, изменять параметры учетной записи и просматривать финансовые сведения.                                         |
| Финансист  | Может просматривать [отчеты о выплатах](payout-summary.md), финансовую информацию и отчеты о приобретениях. Не может изменять приложения, надстройки и параметры учетной записи.                                                                                                                                   |
| Маркетолог             | Может [отвечать на отзывы клиентов](respond-to-customer-reviews.md) и просматривать нефинансовые [аналитические отчеты](analytics.md). Не может изменять приложения, надстройки и параметры учетной записи.      |

В следующей таблице показаны некоторые конкретные функции, доступные каждой из этих ролей (и владельцу учетной записи).

|                                 |    Владелец учетной записи                 |    Менеджер                       |    Разработчик                     |    Бизнес-участник    |    Финансист    |    Маркетолог                      |

|    Отчет о работоспособности                |    Может просматривать                      |    Может просматривать                      |    Может просматривать                      |    Может просматривать                |     Нет доступа             |    Нет доступа                     |

Если ни одна из стандартных ролей не подходит или вы хотите ограничить доступ по определенным приложениям или надстройкам, можно предоставить пользователям собственные разрешения, щелкнув **Настроить разрешения**. Дополнительные сведения см. в разделе [Настройка пользовательских разрешений для пользователей учетной записи](set-custom-permissions-for-account-users.md).

## <a name="add-and-manage-account-users"></a>Добавление пользователей учетной записи и управление ими

Чтобы указать пользователей, которых необходимо добавить в Центр разработки, и назначить им роль, нажмите **Добавить пользователей**.

Вы можете добавить одного или нескольких пользователей из каталога организации в свою учетную запись Центра разработки. Обратите внимание, что при одновременном добавлении нескольких пользователей вы должны назначить им одну и ту же роль. Если вы хотите добавить нескольких пользователей и назначить им разные роли, повторите действия, описанные ниже, для каждой роли.

Кроме того можно предложить пользователям доступ к вашей учетной записи без добавления их в каталог вашей организации.

**Добавление пользователей из каталога организации**

1.  На странице **Управление пользователями** выберите пункт **Добавить пользователей**.
2.  В появившемся списке выберите одного или нескольких пользователей. Чтобы найти определенных пользователей, можно воспользоваться полем поиска.
3.  Выбрав пользователей, нажмите **Добавить выбранные**.
4.  В разделе **Роли** выберите одну или несколько ролей, которые необходимо назначить этому набору пользователей.
5.  Нажмите кнопку **Сохранить**.

Если вы хотите предоставить доступ к Центру разработки новой учетной записи пользователя, можно создать пользователя в разделе **Управление пользователями**. 

По умолчанию переключатель **Добавить в Azure AD** включен. Если оставить его включенным, это создаст новую учетную запись в каталоге вашей организации, и добавит этого пользователя в вашу учетную запись Центра разработки. Если вы не хотите создавать новые учетные записи в каталоге своей организации, но хотите разрешить пользователям получать доступ к вашей учетной записи с помощью их учетных записей Майкрософт, вместо этого выберите **Пригласить пользователей по электронной почте**.

Если необходимо предоставить новому пользователю [Учетную запись глобального администратора](http://go.microsoft.com/fwlink/p/?LinkId=746654) в каталоге вашей организации, установите флажок **Сделать этого пользователя глобальным администратором в Azure AD с полным контролем над всеми ресурсами каталогов**. Это даст пользователю полный доступ ко всем административным функциям в Azure AD вашей компании. Он сможет добавлять пользователей и управлять ими в каталоге вашей организации (но не в Центре разработки, если вы не назначите ему соответствующие [роли и разрешения](#roles-and-permissions). При установке этого флажка необходимо будет предоставить **Электронную почту для восстановления пароля** для этого пользователя.

**Создание новой учетной записи в Центре разработки и в каталоге вашей организации**

1.  На странице **Управление пользователями** выберите пункт **Добавить пользователей**.
2.  На следующей странице нажмите кнопку **Создать пользователя**.
3.  Убедитесь, что переключатель **Добавить в Azure AD** включен.
4.  Введите для нового пользователя имя, фамилию и имя пользователя.
5.  Введите адрес электронной почты, который пользователь сможет использовать, если потребуется восстановить пароль. Это требуется, только если установлен флажок для **Сделать этого пользователя глобальным администратором в Azure AD**.
6.  В разделе **Роли** выберите одну или несколько ролей, которые необходимо назначить новому пользователю, либо задайте пользовательские разрешения.
7.  В разделе **Членство в группах** выберите группы, в которые необходимо добавить нового пользователя.
8.  Нажмите кнопку **Сохранить**.
9.  На странице подтверждения вы увидите учетные данные нового пользователя, в том числе временный пароль. Не забудьте записать эти сведения и предоставить их новому пользователю, так как вы не сможете найти временный пароль, после того как покинете эту страницу.

**Создание новой учетной записи в Центре разработки без добавления пользователя в каталог вашей организации**

1.  На странице **Управление пользователями** выберите пункт **Добавить пользователей**.
2.  На следующей странице нажмите кнопку **Создать пользователя**.
3.  Включите переключатель **Пригласить пользователей по электронной почте**.
3.  Введите один или несколько адресов электронной почты (до 10), разделенные запятыми или точками с запятой.
4.  В разделе **Роли** выберите одну или несколько ролей, которые необходимо назначить новому пользователю, либо задайте пользовательские разрешения.
6.  Нажмите кнопку **Сохранить**.

Приглашенные вами пользователи получат сообщение по электронной почте с приглашением на получение доступа к вашей учетной записи Центра разработки. Каждому пользователю необходимо будет принять свое приглашение, чтобы получить доступ к вашей учетной записи. Для повторной отправки приглашения найдите пользователя на своей странице **Управление пользователями** и щелкните адрес электронной почты (или текст, сообщающий, что **Приглашение требует подтверждения**), чтобы изменить учетную запись. Затем в нижней части страницы нажмите **Повторно отправить приглашение**.

Учетные записи пользователей, добавленные в вашу учетную запись Центра разработки, можно изменять в разделе **Управление пользователями**. Обратите внимание, что имя пользователя и членство в группах будет отражаться как в учетной записи Центра разработки, так и в каталоге вашей организации. Изменение роли пользователя влияет только на доступ в Центре разработки.

**Изменение учетной записи пользователя**

1.  На странице **Управление пользователями** щелкните имя учетной записи пользователя, которую хотите изменить.
2.  Внесите любое из следующих изменений:
    -   Изменение имени, фамилии или имени пользователя. Помните, что эти изменения будут отражены в каталоге вашей организации.
    -   В разделе **Роли** выберите роли, которые вы хотите добавить для пользователя, или снимите выделение для тех ролей, которые необходимо удалить, либо назначьте пользовательские разрешения.
    -   В разделе **Членство в группах** выберите группы, в которые необходимо добавить пользователя, или снимите выделение для тех, из которых необходимо его удалить. Помните, что эти изменения будут отражены в каталоге вашей организации.

3.  Нажмите кнопку **Сохранить**.

Если вы хотите изменить пароль для учетной записи пользователя, добавленной в вашу учетную запись Центра разработки, перейдите в раздел **Управление пользователями**. Обратите внимание, что это действие изменит пароль для каталога пользователя, а не только пароль для доступа к Центру разработки.

**Изменение пароля пользователя для каталога**

Если вы указали **Электронную почту для восстановления пароля** при создании учетной записи пользователя, он сможет изменить свой пароль. Вы можете также изменить пароль пользователя, выполнив указанные ниже действия.

1.  На странице **Управление пользователями** щелкните имя учетной записи пользователя, которую хотите изменить.
2.  Внизу страницы нажмите кнопку **Сбросить пароль**.
3.  Откроется страница подтверждения с учетными данными пользователя и временным паролем.

   > **Важно!** Не забудьте распечатать или скопировать эти сведения и предоставить их пользователю, так как вы не сможете найти временный пароль после того как покинете эту страницу.

## <a name="add-and-manage-groups"></a>Добавление групп и управление ими

При добавлении группы из каталога вашей организации в учетную запись Центра разработки ее члены получают к ней доступ с разрешениями, связанными с назначенной группе ролью. Помните, что любые изменения групп (в том числе ее имени и членов) будут отражены в каталоге организации.

Обратите внимание, что при добавлении нескольких групп одновременно им необходимо назначить одну и ту же роль. Если вы хотите добавить несколько групп и назначить им разные роли, повторите действия, описанные ниже, для каждой роли.

**Добавление групп из каталога вашей организации**

1.  На странице **Управление пользователями** выберите пункт **Добавить группы**.
2.  В появившемся списке выберите одну или несколько групп. Чтобы найти определенные группы, можно воспользоваться полем поиска.
3.  Выбрав группы, нажмите **Добавить выбранные**.
4.  В разделе **Роли** выберите одну или несколько ролей, которые необходимо назначить данному набору групп, либо назначьте пользовательские разрешения.
5.  Нажмите кнопку **Сохранить**.

Если вы хотите предоставить доступ к Центру разработки новой учетной записи группы, можно создать группу в разделе **Управление пользователями**. Обратите внимание, что группа будет создана как в учетной записи Центра разработки, так и в каталоге вашей организации.

**Создание учетной записи группы**

1.  На странице **Управление пользователями** выберите пункт **Добавить группы**.
2.  На следующей странице выберите пункт **Создать группу**.
3.  Введите отображаемое имя для новой группы.
4.  Выберите одну или несколько ролей, которые необходимо назначить новой группе, либо назначьте пользовательские разрешения. Все члены группы будут иметь доступ к вашей учетной записи Центра разработки с разрешениями, связанными с этой ролью.
5.  В появившемся списке выберите одного или нескольких пользователей. Чтобы найти определенных пользователей, можно воспользоваться полем поиска.
6.  Выбрав пользователей, нажмите **Добавить выбранные**.
7.  Нажмите кнопку **Сохранить**.

Учетные записи групп, добавленные в вашу учетную запись Центра разработки, можно изменять в разделе **Управление пользователями**. Обратите внимание, что изменения имени и членов группы будут отражены как в учетной записи Центра разработки, так и в каталоге вашей организации. Изменения роли затронут только доступ этой группы к Центру разработки.

**Изменение учетной записи группы**

1.  На странице **Управление пользователями** выберите имя учетной записи группы, которую хотите изменить.
2.  Чтобы изменить сведения о группе, внесите все необходимые изменения в имя группы. Помните, что эти изменения будут отражены в каталоге вашей организации.
3.  Чтобы изменить роль группы, выберите роли, которые необходимо применить к группе, или отмените выбор ненужных ролей, либо назначьте пользовательские разрешения.
4.  Нажмите кнопку **Сохранить**.

## <a name="add-and-manage-azure-ad-applications"></a>Добавление приложений Azure Active Directory и управление ими

Можно разрешить приложениям или службам, которые являются частью Azure AD вашей организации, получать доступ к вашей учетной записи Центра разработки.

Обратите внимание: при добавлении нескольких приложений Azure Active Directory одновременно им необходимо назначить одну и ту же роль. Если вы хотите добавить несколько групп и назначить им разные роли, повторите действия, описанные ниже, для каждой роли.

**Добавление приложений Azure Active Directory из каталога организации**

1.  На странице **Управление пользователями** выберите пункт **Добавить приложения Azure AD**.
2.  Из появившегося списка выберите одно или несколько приложений Azure Active Directory. Чтобы найти определенные приложения Azure Active Directory, можно воспользоваться полем поиска.
3.  Выбрав приложения Azure Active Directory, нажмите **Добавить выбранные**.
4.  В разделе **Роли** выберите одну или несколько ролей, которые необходимо назначить этому набору приложений Azure AD, либо назначьте пользовательские разрешения.
5.  Нажмите кнопку **Сохранить**.

Если вы хотите предоставить Центру разработки доступ к новой учетной записи приложения Azure AD, можно создать ее в разделе **Управление пользователями**. Обратите внимание: учетная запись будет создана как в учетной записи Центра разработки, так и в каталоге вашей организации.

> **Совет.** Если это приложение Azure AD в основном используется для проверки подлинности Центра разработки и прямой доступ пользователей к нему не требуется, можно ввести любой действительный адрес для параметров **URL-адрес ответа** и **URI кода приложения**, если эти значения не используются никакими другими приложениями Azure AD в вашем каталоге.

**Создание нового приложения Azure AD**

1.  На странице **Управление пользователями** выберите пункт **Добавить приложения Azure AD**.
2.  На следующей странице выберите пункт **Новое приложение Azure AD**.
3.  Введите **URL-адрес ответа** для нового приложения Azure AD. Это URL-адрес, по которому пользователи смогут входить и использовать ваше приложение Azure AD (иногда также называется URL-адресом приложения или URL-адресом входа). Длина **URL-адреса ответа** не может превышать 256 символов.
4.  Введите **URI-адрес идентификатора приложения** для нового приложения Azure AD. Это логический идентификатор приложения Azure AD, который предоставляется, когда оно отправляет запрос единого входа в Azure AD. Обратите внимание: **URI-адрес идентификатора приложения** должен быть уникальным для каждого приложения Azure AD в каталоге и не должен содержать больше 256 символов.
5.  В разделе **Роли** выберите одну или несколько ролей, которые необходимо назначить новому приложению Azure AD, либо назначьте пользовательские разрешения.
6.  Нажмите кнопку **Сохранить**.

После добавления или создания приложения Azure AD можно вернуться в раздел **Управление пользователями** и щелкнуть имя приложения, чтобы просмотреть его параметры, включая идентификатор владельца, идентификатор клиента, URL-адрес ответа и URI кода приложения.

> **Примечание.** Если предполагается использовать API REST, предоставляемые [службами Магазина Windows](../monetize/using-windows-store-services.md), отображаемые на этой странице значения идентификатора владельца и идентификатора клиента потребуются для получения маркера доступа Azure AD, который используется для проверки подлинности при вызовах служб.   

Приложения Azure AD, добавленные в вашу учетную запись Центра разработки, можно изменять в разделе **Управление пользователями**. Обратите внимание: изменения URL-адреса ответа и URI-адреса идентификатора приложения будут отражены как в учетной записи Центра разработки, так и в каталоге вашей организации. Изменения ролей затронут только разрешения приложения Azure AD в Центре разработки.

**Изменение приложения Azure AD**

1.  На странице **Управление пользователями** выберите имя учетной записи приложения Azure AD, которую хотите изменить.
2.  Чтобы изменить **URL-адрес ответа** или **URI-адрес идентификатора приложения**, введите новые значения здесь. Помните, что эти изменения будут отражены в каталоге вашей организации.
3.  Чтобы изменить роль приложения Azure AD, выберите роли, которые необходимо применить, или отмените выбор ненужных ролей либо назначьте пользовательские разрешения.
4.  Нажмите кнопку **Сохранить**.

Если ваше приложение Azure AD считывает и записывает данные в Microsoft Azure AD, ему понадобится ключ. Можно создавать ключи для приложения Azure AD, изменяя его информацию в Центре разработки. Вы можете также удалять ключи, которые больше не используются.

**Управление ключами для приложения Azure AD**

1.  На странице **Управление пользователями** выберите имя приложения Azure AD.

    > **Совет.** Выбрав имя приложения Azure AD, вы увидите все активные ключи для этого приложения, включая дату, когда был создан ключ, и срок окончания его действия. Чтобы удалить ключ, который больше не используется, нажмите **Удалить**.

2.  Чтобы добавить новый ключ, нажмите **Добавить новый ключ**.

3.  Вы увидите экран, на котором отображаются значения **Код клиента** и **Ключ**.

    > **Важно!** Не забудьте распечатать или скопировать эти сведения, так как вы не сможете получить к ним доступ после того, как покинете эту страницу.

4.  Если необходимо создать больше ключей, нажмите **Добавить ключ**.

## <a name="view-history-for-account-users"></a>Просмотр журнала для пользователей учетной записи

Являясь владельцем учетной записи, вы можете просматривать подробную историю браузера для всех дополнительных пользователей, которых вы добавили в учетную запись.

На странице **Управление пользователями** перейдите по ссылке в разделе **Последние действия**, относящейся к пользователю, историю браузера которого требуется просмотреть. Вы можете просматривать URL-адреса всех страниц, которые пользователь посетил за последние 30 дней.

## <a name="removing-users-groups-and-azure-ad-applications"></a>Удаление пользователей, групп и приложений Azure Active Directory

Чтобы удалить пользователя, группу или приложение Azure Active Directory из учетной записи Центра разработки, выберите ссылку **Удалить**, которая появляется рядом с их именами на странице **Управление пользователями**. После подтверждения удаления учетной записи этот пользователь, группа или приложение Azure AD больше не смогут получать доступ к вашей учетной записи Центра разработки (если вы не добавите такой доступ позднее).

> **Примечание.** При удалении пользователь, группа или приложение Azure Active Directory теряют доступ к вашей учетной записи Центра разработки. При этом пользователь, группа или приложение Azure Active Directory не удаляется из каталога вашей организации.
