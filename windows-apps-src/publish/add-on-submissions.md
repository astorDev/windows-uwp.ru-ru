# <a name="add-on-submissions"></a>Отправка надстроек

Надстройки (или продукты в приложении) — это дополнительный элементы для приложения, которые пользователи могут приобретать. Надстройку можно снабдить, новая функция, новая игра уровня, или что-либо еще, вы считаете сохранит пользователей. Надстройки — это не только хороший способ для получения доходов, но и для взаимодействие с клиентами.

Надстройки публикуются с помощью [центра партнеров](https://partner.microsoft.com/dashboard), при этом требуется активная [учетную запись разработчика](https://go.microsoft.com/fwlink/p/?LinkId=615100). Вам также необходимо [включить надстройки](../monetize/in-app-purchases-and-trials.md) в коде приложения.

Первым шагом в процессе отправки надстройки является создание надстройки в центре партнеров по [определения его тип продукта и код продукта](set-your-add-on-product-id.md). После этого вы создадите такие данные, чтобы надстройки можно приобрести через Microsoft Store. Вы можете отправить надстройку одновременно с [самим приложением](app-submissions.md) или работать над ней независимо. Вы также можете выпускать [обновления](#updating-an-add-on-after-publication) надстройки после публикации приложения в Магазине. В этом случае приложение не надо будет отправлять повторно.

> [!NOTE]
> В этом разделе документации описывается отправка надстройки в центре партнеров. Кроме того, с помощью [API отправки в Microsoft Store](../monetize/create-and-manage-submissions-using-windows-store-services.md) можно автоматизировать отправку надстроек.

## <a name="updating-an-add-on-after-publication"></a>Обновление надстройки после публикации

Вы в любое время можете внести изменения в опубликованную надстройку. Отправки изменений надстройку и опубликовано независимо от приложения, поэтому обычно не требуется для обновления всего приложения, чтобы внести изменения в надстройку, такие как обновление ее цена или описание.

Для отправки обновлений, перейдите на страницу add на в центре партнеров и нажмите кнопку **обновления**. Будет создана новая отправка для надстройки, используя в качестве отправной точки информацией, полученной от предыдущей отправки. Внести изменения, например и нажмите кнопку **отправить в Store**.

Если необходимо удалить ранее предложенную надстройку, можно создать новую отправку и изменить значение [Распространение и видимость](set-add-on-pricing-and-availability.md) на **Скрыто в Магазине** с параметром **Остановить приобретение**. Не забудьте обновить код приложения, при необходимости также удалите ссылки на надстройку (особенно в том случае, если приложение ранее опубликованного поддерживает Windows 8.1, более ранней версии; этот параметр видимости не применяется к пользователям ").

> [!IMPORTANT]
> Если ранее опубликованного приложения будет доступен для пользователей Windows 8.x, необходимо будет создать и опубликовать новый отправке приложений, чтобы сделать видимыми для клиентов, обновления надстройки. Аналогичным образом, если вы добавите новые надстройки в ориентированное на Windows 8.x приложение после его публикации, вам понадобится обновить код приложения, чтобы указать ссылки на эти надстройки, а затем повторно отправить приложение. В противном случае новые надстройки не будут видны пользователям в Windows 8.x.
