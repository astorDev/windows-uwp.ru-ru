# <a name="app-submissions"></a>Отправка приложений


После [создания приложения посредством резервирования имени](create-your-app-by-reserving-a-name.md) можно начать процесс его публикации. Первый этап — это создание **отправки**.

Вы можете начать отправку после того, как ваше приложение будет завершено и готово к публикации. Кроме того, можно начать ввод сведений заблаговременно, даже если еще не написано ни единой строчки кода. Сохраняются изменения, внесенные на свое отправление, вы можете вернуться и работать с ним, каждый раз, когда вы будете готовы.

> [!NOTE]
> Необходимо иметь активный [учетную запись разработчика](https://go.microsoft.com/fwlink/p/?LinkId=615100) в [центра партнеров](https://partner.microsoft.com/dashboard) для отправки приложений в Microsoft Store.

После публикации приложения, можно опубликовать обновленной версии, создав другой отправки в центре партнеров. Создание новой отправки позволяет вносить и публиковать любые необходимые изменения будь то при передаче новых пакетов или при простом изменении сведений, например цены или категории. Чтобы создать новый отправки для опубликованного приложения, щелкните **обновления** рядом с последней отправки на его **Обзор** страницы. Вы также можете [удаление приложения из Store](guidance-for-app-package-management.md#removing-an-app-from-the-store) необходимость этого (и затем сделать его доступным еще раз позже, при необходимости).

> [!NOTE]
> В этом разделе документации описывается создание Отправка приложения в центре партнеров. Кроме того, с помощью [API отправки в Microsoft Store](../monetize/create-and-manage-submissions-using-windows-store-services.md) можно автоматизировать отправку надстроек.

> [!IMPORTANT]
> Начиная с 31 октября 2018 года новые продукты не может содержать пакеты, предназначенные для Windows 8.x/Windows Phone 8.x или более ранней версии. Дополнительные сведения см. в этом [блога](https://blogs.windows.com/windowsdeveloper/2018/08/20/important-dates-regarding-apps-with-windows-phone-8-x-and-earlier-and-windows-8-8-1-packages-submitted-to-microsoft-store).
