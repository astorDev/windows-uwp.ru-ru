# <a name="acquisitions-report"></a>Отчет "Приобретения"


Отчет о **приобретениях** панели мониторинга Центр разработчиков Windows позволяет видеть пользователей, которые получит и установленные приложения, а также личных и сведения о платформе и отображает сведения о как прибыли клиентов на 10 Windows (в том числе Xbox) на ваше приложение список. Можно также просмотреть рядом с данными в режиме реального времени приобретения для последнего периода или семьдесят двух часов. 

Можно просмотреть эти данные на информационной панели или [скачать отчет](download-analytic-reports.md), чтобы просмотреть его в автономном режиме. Кроме того, вы можете получить эти данные программным образом с помощью [API REST аналитики Магазина Windows](../monetize/access-analytics-data-using-windows-store-services.md).

В этом отчете **приобретение** означает получение новым клиентом лицензии на ваше приложение (как на возмездной, так и на безвозмездной основе). **Установка** относится к приложению, устанавливаемому на устройстве с Windows 10.

> [!IMPORTANT]
> В отчет **Приобретения** не добавляются данные о возвратах денег, отменах заказов, возвратных платежах ит.д. Чтобы оценить доходы от приложения, просмотрите [сводку о выплатах](payout-summary.md). В разделе **Зарезервировано** перейдите по ссылке **Скачать зарезервированные транзакции**.
>
> За исключением данных о просмотре страницы (как описано ниже), в этом отчете не содержатся данные, связанные с клиентами, которые приобретают приложения, не выполняя вход в учетную запись Майкрософт.


## <a name="apply-filters"></a>Применение фильтров

В верхней части страницы можно выбрать период времени, для которого требуется отобразить данные. Значением по умолчанию является **30D** (30дней), однако вы можете отобразить данные за 3, 6 или 12 месяцев, либо для заданного вручную диапазона дат. Почти в реальном времени данных будут показаны все параметры (кроме данных **накопительный пакет приложения** ). Время **1 H** и **72 H** периоды применяются только на вкладку **приложения ежедневно** диаграммы **приобретения** и перейдите на вкладку **поглощений** **рынков** диаграммы. 

Также можно развернуть раздел **Фильтры**, чтобы отфильтровать все данные на этой странице по рынку или типу устройства.

-   **Рынок**. По умолчанию используется фильтр **Все рынки**, однако вы можете ограничить объем отображаемых данных приобретениями в одном или нескольких ранках.
-   **Тип устройства**. По умолчанию используется фильтр **Все устройства**. Если вам необходимо отобразить данные о приобретениях только с определенного типа устройств (ПК, консоли или планшета), вы можете выбрать его здесь.

Сведения на всех приведенных ниже диаграммах отражают диапазон дат и любые выбранные фильтры. В некоторых разделах можно применять дополнительные фильтры.


## <a name="acquisitions"></a>Приобретения

На диаграмме **Приобретения** отображается количество ежедневных или еженедельных приобретений (то есть число новых клиентов, получающих лицензию на ваше приложение) за выбранный период времени. (При использовании команды **Применить фильтры** для отображения данных за более длительный срок, данные о приобретениях будут сгруппированы по неделям.) Эта диаграмма содержит только те приобретения, которые были сделаны пользователями, выполнившими вход с помощью действительной учетной записи Майкрософт. 

По умолчанию мы показываем представление **ежедневно приложения** , в котором содержит рядом данных реального времени. Вы также можете просмотреть количество покупок для приложения за весь его жизненный цикл, выбрав пункт **Общее количество покупок для приложения**. При этом отображается совокупное общее количество покупок с момента первой публикации приложения.

При необходимости результаты можно отфильтровать по типу магазина, в котором была совершена покупка— магазин клиента или интернет-магазин— либо по версии операционной системы.

> [!NOTE]
> Кроме того, вы можете программным образом получить эти данные с помощью метода [получения сведений о покупках приложения](../monetize/get-app-acquisitions.md) в [API REST аналитики](../monetize/access-analytics-data-using-windows-store-services.md).

В представлении **ежедневно приложения** выбрав **30 D** периода времени, может появиться круг маркеры. Эти представления значительное увеличение или уменьшение в заданное значение, мы считаем, что необходимо знать о. Дата, на котором отображается круг представляет конец недели, в котором обнаружено значительное увеличение или уменьшение по сравнению с недели до этого. Для получения дополнительных сведений об изменениях, наведите указатель мыши на элемент управления circle.  

> [!TIP]
> Вы можете просмотреть дополнительные полезные сведения о связанных с существенные изменения за последние 30 дней [полезные сведения о отчета](insights-report.md).

## <a name="installs"></a>Установки

На диаграмме **Установки** указывается, сколько раз пользователи установили ваше приложение на устройства с Windows 10 (включая консоли Xbox One) за выбранный период времени. На этой диаграмме указывается общее число установок, а также количество установок по дням недели (в зависимости от выбранного периода). При необходимости результаты можно отфильтровать по конкретной версии пакета.

Общее число установок включает в себя:
-   **Установки на нескольких устройствах с Windows 10.** Например, если один и тот же пользователь устанавливает ваше приложение на двух компьютерах с Windows 10 и одной консоли Xbox One, это считается тремя установками.
-   **Повторные установки.** Например, если пользователь устанавливает ваше приложение сегодня, завтра удаляет его и затем снова устанавливает его в следующем месяце, это считается двумя установками.

Сумма установок не включает и не учитывает:
-   **Количество установок на устройства с ОС, отличной от Windows 10.** Если ваше приложение поддерживает более ранние версий ОС, например Windows 8.x или Windows Phone 8.x, установки на этих устройствах не засчитываются.
-   **Удаления.** Когда пользователь удаляет ваше приложение с устройства, это удаление не вычитается из общего числа установок.
-   **Обновления.** Например, если пользователь устанавливает ваше приложение сегодня, а затем устанавливает обновление для приложения через неделю, это считается одной установкой.
-   **Предварительные установки.** Если пользователь покупает устройство, на котором предварительно установлено ваше приложение, это не считается установкой.
-   **Установки, инициированные системой.** Если Windows автоматически устанавливает приложение по какой-либо причине, это не считается установкой.

> [!NOTE]
> Кроме того, вы можете программным образом получить эти данные с помощью метода [получения сведений об установках приложения](../monetize/get-app-installs.md) в [API REST аналитики](../monetize/access-analytics-data-using-windows-store-services.md).

## <a name="acquisition-funnel"></a>Воронка приобретения

На графике **Воронка приобретения** указывается, какое число пользователей выполнило каждый шаг воронки— начиная с просмотра страницы в Магазине и заканчивая использованием приложения, а также коэффициент конверсии. Эти данные помогают выявить направления, которым может потребоваться уделить особое внимание, чтобы увеличить количество приобретений, установок или объем использования.

> [!IMPORTANT]
> На графике **Воронка приобретения** отображаются данные только по пользователям Windows 10 (включая Xbox) за последние 90 дней.

Действия воронки.

- **Просмотры страницы**. Это число представляет собой общее число просмотров страницы описания вашего приложения в Магазине, включая просмотры пользователей, которые не выполнили вход с помощью учетной записи Майкрософт. Эти сведения не включают данные по клиентам, которые отказались предоставить эту информацию корпорации Майкрософт.
- **Приобретения**. Количество новых пользователей, которые получили лицензию на ваше приложение (выполнив вход в систему под своей учетной записью Майкрософт) в течение 48 часов после просмотра его описания в Магазине.
- **Установки**. Число пользователей, установивших приложение после его приобретения.
- **Использование**. Число пользователей, которые использовали приложение после его установки.

При необходимости результаты можно отфильтровать по полу или возрастной группе, а также по идентификатору пользовательской кампании.

> [!NOTE]
> Кроме того, вы можете программным образом получить эти данные с помощью метода [получения сведений о воронке приобретения приложения](../monetize/get-acquisition-funnel-data.md) в [API REST аналитики](../monetize/access-analytics-data-using-windows-store-services.md).

## <a name="markets"></a>Рынки

На диаграмме **Рынки** отображается общее число покупок или установок за выбранный период времени по каждому рынку, в котором доступно ваше приложение. Вы можете указать, как следует отображать данные: для **Приобретений** или для **Установок**.

Эти данные можно просмотреть в визуальной форме **Карта** или в форме **Таблица**. При просмотре данных в табличной форме одновременно отображаются пять рынков с сортировкой в алфавитном порядке или по наибольшему/наименьшему числу приобретений или установок. Также можно скачать данные, чтобы просмотреть сведения для всех рынков сразу.


## <a name="customer-demographic"></a>Демографические сведения о пользователях

Диаграмма **Демографические сведения о пользователях** содержит демографические сведения о людях, которые получили ваше приложение. Вы можете просмотреть, сколько было получений (в течение выбранного периода времени) по определенной возрастной группе и по определенному полу.

> [!NOTE]
> Некоторые пользователи запретили предоставлять эти сведения. Если мы не можем определить возрастную группу или пол, приобретение попадает в категорию **Неизвестно**.

 

## <a name="app-page-views-and-conversions-by-channel"></a>Просмотры и конверсии страницы приложения по каналу

На диаграмме **Просмотры и конверсии страницы приложения по каналу** указывается, каким образом пользователи Windows 10 попали на страницу вашего приложения за выбранный период времени.

На этой диаграмме *канал* означает способ, с помощью которого пользователь попал на страницу с описанием вашего приложения (например, просмотр и поиск в Магазине, ссылка с внешнего веб-сайта, ссылка из одной из пользовательских кампаний и т. д.). Включены следующие типы каналов:

-   **Трафик Магазина:** пользователь выполнял просмотр или поиск в Магазине и увидел описание вашего приложения.
-   **Пользовательская кампания:** пользователь перешел по ссылке, в которой использовался [идентификатор пользовательской кампании](create-a-custom-app-promotion-campaign.md).
-   **Другое:** пользователь перешел по внешней ссылке (без какого-либо идентификатора пользовательской кампании) с веб-сайта в описание вашего приложения, либо пользователь перешел по ссылке из поисковой системы в описание вашего приложения.

*Просмотр страницы* означает, что пользователь просмотрел страницу с описанием вашего приложения в Магазине через Магазин в Интернете или приложение Магазина для Windows 10. К этим данным относятся просмотры пользователей, которые не вошли в систему с помощью учетной записи Майкрософт. Некоторые пользователи отказались предоставить эти сведения корпорации Майкрософт.

*Конверсия* означает, что пользователь (выполнивший вход с помощью учетной записи Майкрософт) получил новую лицензию на ваше приложение (независимо от того, берете ли вы за это деньги или предоставляете лицензию бесплатно).

Число просмотров страницы и данные о конверсии— это не количество уникальных клиентов. Сведения о коэффициенте конверсии см. на диаграмме [Воронка приобретения](#acquisition-funnel).

> [!NOTE]
> Клиенты могли попасть на страницу с описанием вашего приложения, щелкнув пользовательскую кампанию, созданную не вами. Мы помечаем каждый просмотр страницы в сеансе идентификатором кампании, из которой пользователь впервые попал в Магазин. Затем мы присваиваем конверсии этому идентификатору кампании для всех покупок, совершенных в течение 24 часов. По этой причине общее количество конверсий может превышать общее число конверсий для ваших идентификаторов кампаний. Также для конверсий или конверсий надстроек могут отсутствовать просмотры страницы.

> [!NOTE]
> Кроме того, вы можете программным образом получить эти данные с помощью метода [получения сведений о конверсиях приложения по каналу](../monetize/get-app-conversions-by-channel.md) в [API REST аналитики](../monetize/access-analytics-data-using-windows-store-services.md).

## <a name="app-page-views-and-conversions-by-campaign-id"></a>Число просмотров и конверсий страницы приложения по идентификатору кампании

С помощью диаграммы **Число просмотров и конверсий страницы приложения по идентификатору кампании** можно отслеживать конверсии и просмотры страницы для каждой вашей [пользовательской кампании по продвижению на рынок](create-a-custom-app-promotion-campaign.md), как описано выше. На этой диаграмме отображаются идентификаторы самых успешных кампаний. Чтобы отобразить или скрыть конкретные идентификаторы кампаний, можно использовать фильтры.

## <a name="total-campaign-conversions"></a>Общее количество конверсий в ходе кампании

На диаграмме **Общее количество конверсий в ходе кампании** указывается общее число конверсий приложения и надстройки из всех пользовательских кампаний в течение выбранного периода времени.
