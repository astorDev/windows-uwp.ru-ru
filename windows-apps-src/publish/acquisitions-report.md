# <a name="acquisitions-report"></a>Отчет "Приобретения"


**Приобретения** отчет в [центра партнеров](https://partner.microsoft.com/dashboard) позволяет увидеть, кто получит и установки вашего приложения, а также демографические и подробности о платформах и показывает сведения о том, как клиенты, использующие Windows 10 (включая Xbox) попасть в список вашего приложения. Можно также просмотреть практически в реальном времени получения данных за последний период или семьдесят-два часа. 

Эти данные можно просмотреть в центре партнеров или [скачать отчет](download-analytic-reports.md) для автономного просмотра. Кроме того, вы можете получить эти данные программным образом с помощью [API REST аналитики Магазина Windows](../monetize/access-analytics-data-using-windows-store-services.md).

В этом отчете **приобретение** означает получение новым клиентом лицензии на ваше приложение (как на возмездной, так и на безвозмездной основе). **Установка** относится к приложению, устанавливаемому на устройстве с Windows 10.

> [!IMPORTANT]
> **Приобретения** отчет не содержит данных о возмещения, изменение направления, Взимание средств за использование и т. д. Чтобы оценить доход от вашего приложения, посетите [Сводка о выплатах](payout-summary.md). В разделе **Зарезервировано** перейдите по ссылке **Скачать зарезервированные транзакции**.
>
> За исключением данных о просмотре страницы (как описано ниже), в этом отчете не содержатся данные, связанные с клиентами, которые приобретают приложения, не выполняя вход в учетную запись Майкрософт.


## <a name="apply-filters"></a>Применение фильтров

В верхней части страницы можно выбрать период времени, для которого требуется отобразить данные. Значением по умолчанию является **30D** (30 дней), однако вы можете отобразить данные за 3, 6 или 12 месяцев, либо для заданного вручную диапазона дат. Почти в реальном времени данные должны отображаться для всех параметров (за исключением **приложения совокупное** данных). **1H** и **72H** периоды времени применяются только к **ежедневно приложения** вкладке **приобретения** диаграммы и  **Ввод в эксплуатацию** вкладке **рынках** диаграммы. 

Также можно развернуть раздел **Фильтры**, чтобы отфильтровать все данные на этой странице по рынку или типу устройства.

-   **Рынок**: По умолчанию используется фильтр **всех рынков**, но можно ограничить данные для приобретения в один или несколько рынков.
-   **Тип устройства**: Значение по умолчанию — **все устройства**. Если вам необходимо отобразить данные о приобретениях только с определенного типа устройств (ПК, консоли или планшета), вы можете выбрать его здесь.

Сведения на всех приведенных ниже диаграммах отражают диапазон дат и любые выбранные фильтры. В некоторых разделах можно применять дополнительные фильтры.


## <a name="acquisitions"></a>Приобретения

На диаграмме **Приобретения** отображается количество ежедневных или еженедельных приобретений (то есть число новых клиентов, получающих лицензию на ваше приложение) за выбранный период времени. (При использовании команды **Применить фильтры** для отображения данных за более длительный период времени данные будут сгруппированы по неделям.) На этой диаграмме включены только приобретения, сделанных клиентов, которые вошли в систему действительной учетной записи Майкрософт. 

По умолчанию отображаются **приложения ежедневно** представление, которое содержит рядом данных реального времени. Вы также можете просмотреть количество покупок для приложения за весь его жизненный цикл, выбрав пункт **Общее количество покупок для приложения**. При этом отображается совокупное общее количество покупок с момента первой публикации приложения.

**Валовые продажи** для приложения (с октября 2016 г.-присутствует), также доступны на этой диаграмме, показывающий общую сумму денежных средств в результате приложение продаж (в долларах США). Обратите внимание на то, что этот объем не учитывает любые возмещения, изменение направления, о гибком управлении ресурсами и т. д.

При необходимости результаты можно отфильтровать по типу магазина, в котором была совершена покупка — магазин клиента или интернет-магазин — либо по версии операционной системы.

> [!NOTE]
> Кроме того, вы можете программным образом получить эти данные с помощью метода [получения сведений о покупках приложения](../monetize/get-app-acquisitions.md) в [API REST аналитики](../monetize/access-analytics-data-using-windows-store-services.md).

В **ежедневно приложения** представления, когда **30D** выбранного периода времени, вы можете увидеть маркеры круг. Они представляют существенное увеличение или уменьшение заданного значения, мы считаем, что стоит знать о. Дата, на которой отображается элемент управления circle представляет конец недели, в котором мы обнаружили существенное увеличение или уменьшение по сравнению с неделю до этого. Для получения дополнительных сведений об изменениях, наведите указатель на элемент управления circle.  

> [!TIP]
> Можно просмотреть дополнительные сведения, связанные с значительные изменения за последние 30 дней в [отчет Insights](insights-report.md).

## <a name="installs"></a>Установки

На диаграмме **Установки** указывается, сколько раз пользователи установили ваше приложение на устройства с Windows 10 (включая консоли Xbox One) за выбранный период времени. На этой диаграмме указывается общее число установок, а также количество установок по дням недели (в зависимости от выбранного периода). При необходимости результаты можно отфильтровать по конкретной версии пакета.

Общее число установок включает в себя:
-   **Устанавливает на нескольких устройствах Windows 10.** Например, если один и тот же пользователь устанавливает ваше приложение на двух компьютерах с Windows 10 и одной консоли Xbox One, это считается тремя установками.
-   **Переустановка.** Например, если пользователь устанавливает ваше приложение сегодня, завтра удаляет его и затем снова устанавливает его в следующем месяце, это считается двумя установками.

Сумма установок не включает и не учитывает:
-   **Устанавливает на устройствах Windows 10.** Если ваше приложение поддерживает более ранние версий ОС, например Windows 8.x или Windows Phone 8.x, установки на этих устройствах не засчитываются.
-   **Удаляет.** Когда пользователь удаляет ваше приложение с устройства, это удаление не вычитается из общего числа установок.
-   **Обновления.** Например, если пользователь устанавливает ваше приложение сегодня, а затем устанавливает обновление для приложения через неделю, это считается одной установкой.
-   **На ней уже установлены.** Если пользователь покупает устройство, на котором предварительно установлено ваше приложение, это не считается установкой.
-   **Инициированные системой устанавливает.** Если Windows автоматически устанавливает приложение по какой-либо причине, это не считается установкой.

> [!NOTE]
> Кроме того, вы можете программным образом получить эти данные с помощью метода [получения сведений об установках приложения](../monetize/get-app-installs.md) в [API REST аналитики](../monetize/access-analytics-data-using-windows-store-services.md).

## <a name="acquisition-funnel"></a>Воронка приобретения

На графике **Воронка приобретения** указывается, какое число пользователей выполнило каждый шаг воронки — начиная с просмотра страницы в Магазине и заканчивая использованием приложения, а также коэффициент конверсии. Эти данные помогают выявить направления, которым может потребоваться уделить особое внимание, чтобы увеличить количество приобретений, установок или объем использования.

> [!IMPORTANT]
> На графике **Воронка приобретения** отображаются данные только по пользователям Windows 10 (включая Xbox) за последние 90 дней.

Действия воронки.

- **Просмотры страниц**: Это число представляет общее количество просмотров Store пример приложения, включая людей, которые не вошли в систему с учетной записью Майкрософт. Эти сведения не включают данные по клиентам, которые отказались предоставить эту информацию корпорации Майкрософт.
- **Ввод в эксплуатацию**: Число новых клиентов, которые получили лицензию в приложение (если войти в свою учетную запись Майкрософт) в течение 48 часов, просмотра его описание Store.
- **Устанавливает**: Число пользователей, установивших приложения после его получения.
- **Использование**: Число клиентов, которые использовали приложение после его установки.

При необходимости результаты можно отфильтровать по полу или возрастной группе, а также по идентификатору пользовательской кампании.

> [!NOTE]
> Кроме того, вы можете программным образом получить эти данные с помощью метода [получения сведений о воронке приобретения приложения](../monetize/get-acquisition-funnel-data.md) в [API REST аналитики](../monetize/access-analytics-data-using-windows-store-services.md).

## <a name="markets"></a>Рынки

На диаграмме **Рынки** отображается общее число покупок или установок за выбранный период времени по каждому рынку, в котором доступно ваше приложение. Вы можете указать, как следует отображать данные: для **Приобретений** или для **Установок**.

Эти данные можно просмотреть в визуальной форме **Карта** или в форме **Таблица**. При просмотре данных в табличной форме одновременно отображаются пять рынков с сортировкой в алфавитном порядке или по наибольшему/наименьшему числу приобретений или установок. Также можно скачать данные, чтобы просмотреть сведения для всех рынков сразу.


## <a name="customer-demographic"></a>Демографические сведения о пользователях

Диаграмма **Демографические сведения о пользователях** содержит демографические сведения о людях, которые получили ваше приложение. Вы можете просмотреть, сколько было получений (в течение выбранного периода времени) по определенной возрастной группе и по определенному полу.

> [!NOTE]
> Некоторые пользователи запретили предоставлять эти сведения. Если мы не можем определить возрастную группу или пол, получение попадает в категорию **Неизвестно**.

 

## <a name="app-page-views-and-conversions-by-channel"></a>Представления и преобразования страницы приложения по каналу

**Просмотры страниц приложения и преобразования каналом** диаграммы позволяет увидеть, как клиенты, использующие Windows 10 будет достигать листинг вашего приложения за выбранный период времени.

На этой диаграмме *канал* означает способ, с помощью которого пользователь попал на страницу с описанием вашего приложения (например, просмотр и поиск в Магазине, ссылка с внешнего веб-сайта, ссылка из одной из пользовательских кампаний и т. д.). Включены следующие типы каналов:

-   **Трафик Store.** Просмотр или поиск в Store, при их просмотре вашего приложение — листинг клиента.
-   **Пользовательские кампании:** Клиент, а затем ссылку, которая используется [идентификатор пользовательской кампании](create-a-custom-app-promotion-campaign.md).
-   **Другие:** Клиент за веб-сайта в приложение — листинг следует внешней ссылки (без кода пользовательской кампании) или клиента перешел от механизм поиска к списку в приложения.

Объект *просмотра страницы* означает, что клиенту просмотреть страницу с описанием вашего приложения Store, либо через веб-Store, либо из app Store в Windows 10. К этим данным относятся просмотры пользователей, которые не вошли в систему с помощью учетной записи Майкрософт. Некоторые пользователи отказались предоставить эти сведения корпорации Майкрософт.

*Конверсия* означает, что пользователь (выполнивший вход с помощью учетной записи Майкрософт) получил новую лицензию на ваше приложение (независимо от того, берете ли вы за это деньги или предоставляете лицензию бесплатно).

Число просмотров страницы и данные о конверсии — это не количество уникальных клиентов. Сведения о коэффициенте конверсии см. на диаграмме [Воронка приобретения](#acquisition-funnel).

> [!NOTE]
> Клиенты могли попасть на страницу с описанием вашего приложения, щелкнув пользовательскую кампанию, созданную не вами. Мы помечаем каждый просмотр страницы в сеансе идентификатором кампании, из которой пользователь впервые попал в Магазин. Затем мы присваиваем конверсии этому идентификатору кампании для всех покупок, совершенных в течение 24 часов. По этой причине общее количество конверсий может превышать общее число конверсий для ваших идентификаторов кампаний. Также для конверсий или конверсий надстроек могут отсутствовать просмотры страницы.

> [!NOTE]
> Кроме того, вы можете программным образом получить эти данные с помощью метода [получения сведений о конверсиях приложения по каналу](../monetize/get-app-conversions-by-channel.md) в [API REST аналитики](../monetize/access-analytics-data-using-windows-store-services.md).

## <a name="app-page-views-and-conversions-by-campaign-id"></a>Число просмотров и преобразований страницы приложения по идентификатору кампании

С помощью диаграммы **Число просмотров и конверсий страницы приложения по идентификатору кампании** можно отслеживать конверсии и просмотры страницы для каждой вашей [пользовательской кампании по продвижению на рынок](create-a-custom-app-promotion-campaign.md), как описано выше. На этой диаграмме отображаются идентификаторы самых успешных кампаний. Чтобы отобразить или скрыть конкретные идентификаторы кампаний, можно использовать фильтры.

## <a name="total-campaign-conversions"></a>Общее количество конверсий в ходе кампании

На диаграмме **Общее количество конверсий в ходе кампании** указывается общее число конверсий приложения и надстройки из всех пользовательских кампаний в течение выбранного периода времени.
