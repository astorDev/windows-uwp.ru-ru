# <a name="configure-precise-release-scheduling"></a>Настройка точного планирования выпуска

В разделе **Расписание** на странице [цены и доступности](set-app-pricing-and-availability.md) можно задать точную дату и время, когда приложение должно быть доступно в магазине, что обеспечивает большую гибкость и возможность настройки дат для различных рынков.

> [!NOTE]
> Хотя этот раздел относится к приложениям, планирование выпуска для отправки надстройки использует тот же процесс.

Вы также можете указать дату, когда продукт больше не должен быть доступен в магазине. Обратите внимание, что это означает, что продукт больше не может быть найден в магазине с помощью поиска или просмотра, но любой клиент с прямой ссылкой может просмотреть список магазинов продукта. Они могут скачать его только в том случае, если они уже владеют продуктом или имеют [рекламный код](generate-promotional-codes.md) и используют устройство Windows 10.

По умолчанию (если не выбрано одно из действий **сделать это приложение доступным, но не обнаруживаемым в** параметрах магазина в разделе [видимость](choose-visibility-options.md#discoverability) ), ваше приложение будет доступно клиентам сразу после того, как оно прошло сертификацию и завершит публикацию. процесс. Чтобы выбрать другие даты, щелкните **Показывать параметры** , чтобы развернуть этот раздел.

Обратите внимание, что вы не сможете настроить даты в разделе " **Расписание** ", если вы выбрали одно из действий **сделать это приложение доступным, но не обнаруживаемое в** параметрах магазина в разделе [видимость](choose-visibility-options.md#discoverability) , так как ваше приложение не будет выпущено Клиенты, поэтому нет даты выпуска для настройки.

> [!IMPORTANT]
> Даты, указанные в разделе Расписание, применяются только к клиентам в Windows 10.
>
>Если ранее опубликованное приложение поддерживает более ранние версии ОС, любая выбранная дата **приобретения** не будет применяться к этим клиентам. они по-прежнему смогут получить приложение (если только вы не отправите обновление с новым выбором в разделе [видимость](choose-visibility-options.md#discoverability) или если на странице **Обзор приложения** выбран вариант **сделать приложение недоступным** ).


## <a name="base-schedule"></a>Базовое расписание

Выбор, сделанный для базового расписания, будет применяться ко всем рынкам, в которых доступно приложение, если в дальнейшем не добавить даты для конкретных рынков (или групп рынка), выбрав [настроить для конкретных рынков](#customize-the-schedule-for-specific-markets).

Вы увидите два варианта: **выпуск** и **прекратить приобретение**. 

## <a name="release"></a>Release

В раскрывающемся списке **Release (выпуск** ) можно задать, когда приложение должно быть доступно в магазине. Это означает, что приложение может быть обнаружено в магазине с помощью поиска или просмотра, и клиенты могут просматривать свои списки магазинов и получать приложения.

>[!NOTE]
> После публикации приложения и его доступности в магазине вы больше не сможете выбрать дату **выпуска** (так как приложение уже будет выпущено).

Ниже приведены параметры, которые можно настроить для расписания **выпуска** продукта.
- **как можно скорее**: продукт будет выпущен, как только он будет сертифицирован и опубликован. Это вариант по умолчанию.
- **в**: продукт будет выпущен на выбранную дату и время. Кроме того, у вас есть два варианта:
   - **UTC**. выбранное время будет иметь значение всеобщего скоординированного времени (UTC), чтобы выпуск приложения находился в одно и то же время.
   - **Локальный**. выбранное время будет использоваться в каждом часовом поясе, связанном с рынком. (Обратите внимание, что для рынков, включающих более одного часового пояса, будет использоваться только один часовой пояс на этом рынке. Для США используется Восточного часового пояса. Полный список часовых поясов приведен ниже на этой странице.)
- **не запланировано**: приложение будет недоступно в магазине. Если выбран этот параметр, можно сделать приложение доступным в хранилище позже, создав новую отправку и выбрав один из других параметров.


## <a name="stop-acquisition"></a>Прерывать получение

В раскрывающемся списке " **закончить получение** " можно задать дату и время, когда вы хотите запретить новым клиентам получать его из магазина или обнаруживать его список. Это может быть полезно, если нужно точно контролировать, когда приложение больше не будет предлагаться новым клиентам, например, когда вы собираетесь обеспечить доступность между несколькими приложениями.

По умолчанию для параметра **прерывать получение** установлено значение никогда. Чтобы изменить это значение, **выберите в** раскрывающемся списке пункт в и укажите дату и время, как описано выше. После выбора даты и времени клиенты больше не смогут получить приложение.

Важно понимать, что этот параметр имеет такое же влияние, как и выбор **сделать это приложение обнаруживаемым, но недоступным** в разделе " [видимость](choose-visibility-options.md#discoverability) ", и выбрав пункт " **прекращение приобретения": любой клиент с прямой ссылкой может видеть магазин продукта. , но они могут скачать его только в том случае, если они принадлежали продукту, или имеют рекламный код и используют устройство Windows 10.** Чтобы полностью отключить предложение приложения новым клиентам, щелкните **сделать приложение недоступным** на странице Обзор приложения. Дополнительные сведения см. [в разделе Удаление приложения из магазина](guidance-for-app-package-management.md#removing-an-app-from-the-store).

> [!TIP]
> Если вы выбрали дату для **отмены приобретения**и позже решили снова сделать приложение доступным, вы можете создать новую отправку **и изменить значение "больше"** **.** После публикации обновленной отправки приложение снова станет доступным.

## <a name="customize-the-schedule-for-specific-markets"></a>Настройка расписания для конкретных рынков 

По умолчанию выбранные выше параметры будут применяться ко всем рынкам, в которых предлагается ваше приложение. Чтобы настроить цену для конкретных рынков, щелкните **настроить для конкретных рынков**. Появится всплывающее окно **Выбор рынка** , в котором перечислены все рынки, в которых вы выбрали, чтобы сделать приложение доступным. Если вы исключили все рынки в разделе [рынки](define-pricing-and-market-selection.md) , эти рынки отображаться не будут. 

Чтобы добавить расписание для одного рынка, выберите его и нажмите кнопку **сохранить**. Вы увидите те же параметры **выпуска** и **отмены приобретения** , описанные выше, но сделанные вами варианты будут применены только к этому рынку.

Чтобы добавить расписание, которое будет применяться к нескольким рынкам, создайте *группу рынка*. Для этого выберите рынки, которые требуется включить, а затем введите имя группы. (Это имя предназначено только для справки и не будет отображаться для клиентов.) Например, если вы хотите создать группу рынка для Северная Америка, можно выбрать пункт **Канада**, **Мексика**и **США**и назовите его **Северная Америка** или другое имя. Завершив создание группы рынка, нажмите кнопку **сохранить**. Вы увидите те же параметры **выпуска** и **отмены приобретения** , описанные выше, но сделанные вами варианты будут применены только к этой группе рынка.

Чтобы добавить пользовательское расписание для дополнительного рынка или дополнительную группу рынка, просто щелкните **настроить для конкретных рынков** еще раз и повторите эти действия. Чтобы изменить рынки, входящие в группу рынка, выберите ее имя. Чтобы удалить настраиваемое расписание для группы рынка (или отдельного рынка), нажмите кнопку **Удалить**.

> [!NOTE]
> На рынке не могут входить более одной из групп рынка, используемых в разделе " **Расписание** ".
