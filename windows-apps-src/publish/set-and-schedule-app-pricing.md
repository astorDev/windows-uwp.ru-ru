# <a name="set-and-schedule-app-pricing"></a>Настройка и планирование цены приложения

Раздел **Цена** на странице [Цены и доступность](set-app-pricing-and-availability.md) позволяет выбрать базовую цену для приложения. Вы также можете [запланировать изменение цены](#schedule-price-changes), указав дату и время, когда цена вашего приложения должна измениться. Кроме того, вы можете [настроить эти изменения для отдельных рынков](#customize-pricing-for-specific-markets). 

> [!NOTE]
> Несмотря на то что этот раздел относится к приложениям, выбор цены для отправок надстроек выполняется аналогичным образом.

## <a name="base-price"></a>Базовая цена

При выборе **базовой цены** для вашего приложения эта цена будет использоваться на любом рынке, где будет продаваться ваше приложение, если только вы не укажете особую цену для отдельных рынков.

Вы можете задать для **базовой цены** значение **Бесплатно** или выбрать доступную ценовую категорию, которая определяет цену продажи во всех странах, где вы распространяете приложение. Ценовые категории начинаются с 0,99долларов США и далее увеличиваются пошагово (1,09; 1,19 долларов США и т.д.). Как правило, величина шага также растет с увеличением цены. 

> [!NOTE]
> Эти ценовые категории также применимы к надстройкам. 

Каждая ценовая категория имеет соответствующее значение в одной из более чем 60 валют, которые поддерживает Магазин. Мы используем эти значения, чтобы помочь вам продавать приложения по всему миру по сопоставимой цене. Вы можете выбрать базовую цену в любой валюте, и мы будем использовать соответствующие цены для других рынков.

В разделе **Цена** нажмите **Просмотреть таблицу**, чтобы просмотреть соответствующие цены во всех валютах. В ней также отображается идентификационный номер, связанный с каждой ценовой категорией, который понадобится вам, если вы используете [API отправки в Магазин Windows](../monetize/manage-app-submissions.md#price-tiers) для ввода цен. Вы можете нажать **Скачать**, чтобы скачать копию таблицы ценовых категорий как CSV-файл.

Помните, что выбранный набор цен может включать в себя налог с продажи или налог на добавленную стоимость, которые должны оплачиваться вашими клиентами. Дополнительные сведения о налоговых вопросах на отдельных рынках см. в разделе [Налогообложение для платных приложений](tax-details-for-paid-apps.md). Также ознакомьтесь с [факторами, которые необходимо учитывать при выборе цен для отдельных рынков](define-pricing-and-market-selection.md#price-considerations-for-specific-markets).

## <a name="schedule-price-changes"></a>Планирование изменения цен

Вы можете запланировать изменение цен, если хотите, чтобы базовая цена приложения изменилась в определенное время. 

> [!IMPORTANT]
> Изменения цены отображаются только для пользователей Windows 10. Если ваше приложение поддерживает более ранние версии ОС, цены не будут меняться. 
>
> - Пользователям Windows 8 приложение всегда будет предлагаться по **базовой цене** (а не особой цене для рынка), даже если вы запланируете изменения цены. 
> - Пользователям Windows 8.1 и Windows Phone 8.1 и более ранних версий приложение всегда будет предлагаться по исходной цене для рынка пользователя, даже если вы запланируете изменения цены на этом рынке.
> 
> Помните об этом, планируя изменения цены. Например, если вы выпустите приложение по более низкой цене, а затем запланируете повышение цены, пользователи более ранних версий ОС при покупке будут платить за него более низкую (исходную) цену.

Нажмите **Запланировать изменение цены**, чтобы открыть параметры изменения цены. Выбор нужную ценовую категорию, а затем дату, время и часовой пояс.

Вы можете нажать **Запланировать еще одно изменение цены**, чтобы запланировать еще одно изменение.

> [!NOTE]
> Запланированные изменения цен работают не так, как [цена распродажи](put-apps-and-add-ons-on-sale.md). Когда вы назначаете распродажу, пользователи, просматривающие описание в Магазине, увидят, что цена снижена, и смогут приобрести приложение по более низкой цене в течение выбранного вами периода. После окончания периода распродажи вернется базовая цена.
>
> Запланированное изменение цены позволяет скорректировать цену. Изменение вступит в силу в указанную дату, но не будет отображаться как распродажа в Магазине. У приложения только будет новая базовая цена. 

## <a name="customize-pricing-for-specific-markets"></a>Настройка цены для отдельных рынков

По умолчанию выбранные выше варианты будут применены для всех стран, в которых доступно приложение. Чтобы изменить цену для определенных рынков, выберите **Настроить для определенных рынков**. Появится всплывающее окно **Выбор страны** с перечислением всех выбранных вами стран, в которых доступно ваше приложение. Страны, исключенные в разделе "Страны", не отображаются здесь. 

> [!IMPORTANT]
> Пользователи Windows 8 всегда будут видеть **базовую цену**, даже если вы выберите другую цену для их рынка.

Чтобы добавить особую цену для рынка, выберите ее и нажмите **Сохранить**. После этого будут отображаться те же параметры **Базовая цена** и **Запланировать изменение цены**, что описано выше, но для этого рынка будет применяться выбранная цена.

Чтобы добавить особую цену для нескольких рынков, создайте *группу рынков*. Чтобы сделать это, выберите страны, которые вы хотите включить в группу, а затем введите имя группы. (Это имя предназначено только для вас и не будет видно пользователям.) Когда закончите, нажмите **Сохранить**. После этого будут отображаться те же параметры **Базовая цена** и **Запланировать изменение цены**, что описано выше, но для этой группы рынков будет применяться выбранная цена.

> [!NOTE]
> Тот или иной рынок может быть включен только в одну группу рынков, которые используются в разделе "Цена".

Чтобы добавить особую цену для другого рынка или группы рынков, выберите **Настроить для определенных рынков** снова и повторите описанную процедуру. Чтобы изменить страны, включенные в группу, выберите ее имя. Чтобы удалить цену для группы рынков (или отдельного рынка), нажмите **Удалить**.
