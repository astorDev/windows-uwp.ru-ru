# Определение цены и выбор рынка


Магазин Windows доступен для потребителей в более чем 200 странах и регионах по всему миру. Вы можете выбирать рынки, на которых следует предлагать ваше приложение, а также можете определить цену, которую должны будут платить пользователи. У вас также есть возможность определять цены в зависимости от рынка.

**Примечание.** Ознакомьтесь с нашими [рекомендациями по глобализации приложений](https://msdn.microsoft.com/library/windows/apps/hh969150).

 

## Базовая цена


При выборе **базовой цены** для вашего приложения эта цена будет использоваться на любом рынке, где будет продаваться ваше приложение, если только вы не укажете особую цену для определенного рынка.

Можно установить для параметра **Базовая цена** значение **Бесплатно**, а также использовать доступную ценовую категорию. устанавливает цену продажи во всех странах, где вы хотите распространять приложение. Ценовые категории начинаются с 0,99 долларов США и далее увеличиваются (1,29; 1,49; 1,99 долларов США и т. д.). Величина шага также растет с увеличением цены.

> **Примечание.** Эти ценовые категории также применимы к [внутренним продуктам приложений (IAP)](iap-submissions.md).

Каждая ценовая категория имеет соответствующее значение в одной из более чем 60 валют, которые поддерживает Магазин. Мы используем эти значения, чтобы помочь вам продавать приложения по всему миру по сопоставимой цене. Однако из-за колебаний в курсах обмена валют точная сумма продажи может немного меняться от валюты к валюте.

Помните, что выбранный набор цен может включать в себя налог с продажи или налог на добавленную стоимость, которые должны оплачиваться вашими клиентами. Дополнительные сведения о налоговых вопросах на отдельных рынках см. в разделе [Налогообложение для платных приложений](tax-details-for-paid-apps.md). См. также [Вопросы цен на отдельных рынках](#price-considerations-for-specific-markets) ниже.

## Рынки и особые цены


По умолчанию ваше приложение будет отображаться по базовой цене на всех возможных рынках, включая любые последующие рынки, которые могут быть добавлены позже.

Чтобы изменить эти параметры, разверните раздел **Рынки и особые цены** на странице **Цена и доступность** отправки вашего приложения. Можно отменить выбор отдельных рынков для их исключения или щелкнуть **Отменить все**, а затем добавить отдельные рынки по своему выбору.

> **Важно!** Вы отвечаете за соблюдение местных нормативных требований, даже если эти требования не указаны здесь или на информационной панели Центра разработки для Windows.

Если вы хотите установить особые цены на ваше приложение для определенных рынков, это также можно сделать здесь; в противном случае по умолчанию ваше приложение будет предоставляться по базовой цене (в локальной валюте рынка).

> **Примечание.** Можно также определить особые цены для определенных рынков при [отправках IAP](iap-submissions.md).

Следует учитывать, что даже если вы выберете все рынки, местные законы, ограничения и прочие факторы могут помешать включению в каталог некоторых приложений для некоторых стран и регионов. Кроме того, на некоторых рынках могут предъявляться особые требования в отношении возрастных категорий. Если приложение не соответствует этим требованиям, мы не сможем опубликовать ваше приложение на этом рынке. Дополнительные сведения см. в разделе [Возрастные категории](age-ratings.md).

> **Примечание.** Для приложений, предназначенных для Windows 8 или Windows 8.1, некоторые отдельные рынки обрабатываются как единый рынок «Остальной мир». Подробнее: [Рынки «Остальной мир» для Windows 8.x](#rest-of-world-markets-for-windows-8-x). Также помните, что для пользователей устройств Windows 8 или Windows 8.1 приложение будет отображаться только по базовой цене, даже если вы изменили цену на их рынке.

## Вопросы цен на отдельных рынках


Такие способы оплаты, как карты оплаты и выставление счетов через мобильного оператора, позволяют повысить продажи платных приложений и покупок из приложений. Из-за более высоких расходов на подобные способы оплаты к комиссии Магазина, вычитаемой из чистой выручки, применяется рекламная поправка. Это позволяет определить сумму выплат за платные приложения и продукты в приложениях в странах и регионах, где используются перечисленные в таблицах ниже способы оплаты. Вам стоит решить, применима ли рекламная поправка в стране или регионе, где продается ваше приложение, и учесть этот фактор в своей рыночной стратегии. Дополнительные сведения о рекламной поправке можно найти в [Соглашении разработчика приложений](https://msdn.microsoft.com/library/windows/apps/hh694058).

Рекламная поправка применяется ко всем транзакциям, обрабатываемым в указанной стране или регионе по выбранным способам оплаты, начиная с даты вступления в силу. Эти сведения обновляются ежемесячно. Новые страны, регионы и способы оплаты будут добавляться в течение 30 суток после начала действия рекламной поправки для этой страны или региона и способа оплаты.

## Рынки «Остальной мир» для Windows 8.x


Если ваше приложение включает пакеты для Windows 8.x, важно понимать, что некоторые рынки обрабатываются как единый рынок «Остальной мир» для пользователей Магазина на устройствах Windows 8.x, даже если они теперь отображаются как отдельные рынки на информационной панели Центра разработки для Windows (в отличие от более ранней информационной панели Магазина Windows, где можно было выбрать только вариант рынка «Остальной мир»).

Если вы оставите выбор по умолчанию при отправке приложения, то вам не о чем беспокоиться, ваше приложение будет доступно на всех возможных рынках. Если же вы хотите исключить некоторые рынки, помните, что исключение даже одного из этих рынков «Остального мира» означает, что ваше приложение будет недоступно на всех рынках «Остального мира» для пользователей Windows 8 или Windows 8.1.

Рынки, которые входят в «Остальной мир» для Windows 8.x:

-   Афганистан
-   Аландские острова
-   Албания
-   Американское Самоа
-   Андорра
-   Ангола
