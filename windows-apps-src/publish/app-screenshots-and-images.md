# Снимки экрана и изображения приложения


Ваше приложение будет содержать различные логотипы, снимки экрана и другие изображения. Некоторые из них обязательны, некоторые — нет. Помните, что изображения — один из основных способов представить ваше приложение. Хорошо продуманные изображения сделают приложение более привлекательным для пользователя.

В [процессе отправки приложения](app-submissions.md) вы предоставляете [снимки экрана](#screenshots) и [рекламное изображение](#promotional-artwork) на шаге [Описания](create-app-descriptions.md). Эти изображения используются для демонстрации вашего приложения в Магазине.

Магазин также использует плитку вашего приложения и другие изображения, которые включены в пакет вашего приложения. Запустите [комплект сертификации приложений для Windows](https://msdn.microsoft.com/library/windows/apps/mt186449), чтобы определить, не отсутствуют ли какие-либо обязательные изображения. Инструкции и рекомендации, связанные с этими изображениями, см. в статье [Ресурсы плиток и значков](../controls-and-patterns/tiles-and-notifications-app-assets.md).

> **Примечание**  Способы использования изображений в Магазине, на начальном экране пользователя и в самом приложении могут различаться в зависимости от операционной системы пользователя и других факторов.

## Снимки экрана

Снимки экрана — это изображения вашего приложения, которые пользователи смогут просмотреть в каталоге Магазина.

Вы увидите несколько полей на страницу **Описание**, где вы можете добавить снимки экрана для различных семейств устройств (которые будут отображаться, когда пользователь будет просматривать описание вашего приложения в Магазине на устройстве соответствующего типа).

Для отправки требуется только один снимок экрана (хотя можно предоставить несколько снимков — до 9 снимков экрана с настольного устройства и до 8 снимков экрана с мобильных и голографических устройств). Вам не обязательно предоставлять отдельные снимки экрана для каждого семейство устройств, но мы рекомендуем предоставлять снимки экрана для каждого типа, поддерживаемого вашим приложением, чтобы клиенты могли получить представление о том, как это приложение будет выглядеть на их устройстве.

> **Примечание**  В состав Microsoft Visual Studio входит [средство для создания снимков экрана](http://go.microsoft.com/fwlink/p/?LinkId=221135).

Каждый снимок экрана должен представлять собой PNG-файл в альбомной или книжной ориентации, а размер файла не может превышать 2 МБ.

Требования к размеру зависят от семейства устройств.
- Мобильное: 768 x 1280, 720 x 1280 или 480 x 800 пикселей
- Настольное: 1366 x 768 пикселей или больше
- Голографическое: 1268 x 720 пикселей или больше

Под каждым снимком экрана можно оставить краткую подпись длиной не больше 200 символов.

> **Примечание**  Если вы создадите описания для [нескольких языков](supported-languages.md), для каждого из них будет использоваться своя страница **Описание**. Вам потребуется загрузить изображения для каждого языка по отдельности (даже при использовании тех же изображений) и включить подписи для каждого языка.


## Значок плитки приложения

Этот элемент не является обязательным для всех отправок, но он настоятельно рекомендуется, если ваше приложение работает в ОС Windows Phone 8.1 или более ранних версий. Значок плитки приложения используется во время отображения описания вашего приложения для клиентов в ОС Windows Phone 8.1 и более ранних версий. Если вы не предоставите это изображение, клиенты, использующие Windows Phone 8.1 или более ранних версий, будут видеть пустой значок в описании вашего приложения. (Это также относится к клиентам, использующим Windows 10, если в вашем приложении есть только пакеты, предназначенные для Windows Phone 8.1 или более ранних версий.)

В качестве значка плитки приложения необходимо использовать PNG-файл размером 300 x 300 пикселей.

## Рекламное изображение

Редакция Магазина Windows использует различные изображения для представления приложений в Магазине. Отправка рекламного изображения позволяет команде Магазина Windows учитывать ваше приложение в рекламных макетах.

> **Важно**  Предоставление этих рекламных изображений для приложения не гарантирует, что оно будет включено в подборку, но если их нет, то возможность рекламы рассматриваться не будет. (Подробнее см. в разделе [Повышение популярности вашего приложения](make-your-app-easier-to-promote.md).)

Вы можете отправить рекламное изображение в разных размерах, в зависимости от версий ОС, для которых предназначено ваше приложение. Независимо от размера, все изображения должны быть в формате PNG.

Вот несколько советов, о которых следует помнить при разработке рекламного изображения.

-   Выбирайте динамические изображения, которые имеют отношение к приложению и помогают быстрее его узнать и выделить среди других. Не используйте изображения из фотобанков или обобщенные визуальные элементы.
-   Не включайте текст (кроме вашей фирменной символики).
-   Сведите пустое пространство на изображении к минимуму.
-   Старайтесь не отображать пользовательский интерфейс вашего приложения и не используйте изображения, относящиеся к конкретному устройству.
-   Избегайте политических и национальных тем, флагов и религиозных символов.
-   Не включайте изображения нетактичных жестов, изображения обнаженного тела, азартных игр, валюты, наркотиков, табака или алкоголя.
-   Не используйте элементы, изображающие наведение оружия на пользователя или чрезмерное насилие и его последствия.

### Для Windows 10: 2400 x 1200

В Магазине для Windows 10 в верхней части страниц категории «Приложения и игры» для продвижения содержимого имеется вращающееся изображение «В центре внимания». Чтобы ваше приложение было размещено в центре внимания, обязательно отправьте изображение размером 2400 x 1200.

При разработке изображения учитывайте то, что в случае размещения в категории «В центре внимания» будет применяться градиент в нижней трети изображения, необходимый для наложения поверх него различимого рекламного текста. Поэтому не размещайте свой текст и основные визуальные элементы в нижней трети. Кроме того, мы можем обрезать изображение, поэтому фирменную символику и наиболее важные детали следует размещать в центре изображения.

На рисунке ниже изображены основные пропорции, которые следует учитывать. «Защищенная зона» в центре будет видимой, даже если мы обрежем изображение. «Динамическая область текста» — это область, в которой может появиться текст и градиент.

### Для Windows Phone 8.1 и более ранних версий: 1000 x 800, 358 x 358

В Магазине для Windows Phone 8.1 и более ранних версий в пропорциональных макетах можно использовать два размера изображения: 1000 x 800 и 358 x 358 пикселей. Если ваше приложение работает в Windows Phone 8.1 или более ранних версий, мы рекомендуем предоставить изображения двух размеров для рекламных целей.

> **Совет**   Кроме того, предоставьте [изображение значка плитки приложения](#app-tile-icon) размером 300 x 300, чтобы ваше приложение не отображалось в Магазине с пустым значком.

### Для Windows 8.1 и более ранних версий: 414 x 180

В Магазине для Windows 8.1 и более ранних версий для рекламных макетов могут использоваться изображения размером 414 x 180 пикселей. Если приложение работает в Windows 8.1 или более ранних версий, мы рекомендуем предоставить изображение этого размера для рекламных целей.
