# <a name="beta-testing-and-targeted-distribution"></a>Бета-тестирование и целевое распространение

Даже самое тщательное тестирование приложения не сравнится с реальной проверкой другими пользователями. Информационная панель Центра разработки для Windows позволяет предоставлять доступ к отправке приложения только для определенных людей, например для тестировщиков, чтобы они проверили работу приложения перед его публикацией. Бета-тестеры могут обнаружить проблемы, которые вы пропустили, например, орфографические ошибки, неясную работу приложения или ошибки, которые могут привести к сбоям приложения. У вас будет возможность исправить эти проблемы, прежде чем сделать отправленное приложение общедоступным, и в результате вы получите более доработанный продукт.

Мы предоставляем несколько способов ограничить область распространения приложений только тестировщиками, не создавая отдельную версию приложения с другим именем и идентификатором пакета. Разумеется, если требуется, вы можете создать отдельное приложение для тестирования. В таком случае присвойте ему имя, которое отличается от окончательного публичного имени приложения.

Метод распространения приложения тестировщикам зависит от операционных систем, для которых предназначено ваше приложение. Ниже перечислены варианты, актуальные для Windows10, а также Windows Phone 8.1 и более ранних версий.

## <a name="making-your-app-available-to-testers-on-windows-10-devices"></a>Предоставление доступа к вашему приложению тестировщикам на устройствах Windows 10

Мы предоставляем два способа ограничить область распространения приложений только определенными пользователями с устройствами Windows 10.

### <a name="package-flights"></a>Тестовые пакеты

Если вы уже опубликовали свое приложение, вы можете создавать тестовые пакеты, чтобы распространять указанным пользователям другие наборы пакетов. Вы даже можете создать несколько тестовых пакетов одного приложения для разных групп пользователей. Это отличный способ испытать несколько пакетов одновременно, который позволяет переносить тестируемые возможности в приложение в случае обычной отправки, если вы решите, что пакеты можно публиковать для всех.

Дополнительные сведения можно найти в разделе [Тестовые пакеты](package-flights.md).

> [!NOTE]
> Для распространения конкретных пакетов среди случайной выборки пользователей Windows10 (с указанием количества в процентах), а не среди заданной группы пользователей, можно использовать [постепенный выпуск пакетов](gradual-package-rollout.md). Можно также объединить выпуск с тестовыми пакетами, если требуется постепенно распространять обновление в одной из тестовых групп.

### <a name="hiding-the-app-in-the-store-and-using-promotional-codes"></a>Скрытие приложения в Магазине и использование рекламных кодов

Если вам требуется ограничить область распространения приложения определенной группой тестировщиков, **не** публикуя общедоступную отправку, вы можете использовать тот же [процесс отправки приложения](app-submissions.md), что и для остальных приложений. Чтобы разрешить только определенным пользователям получать приложение бесплатно, а также скрыть описание приложения от других пользователей и запретить им скачивать его, сделайте следующее:

-   В своей отправке в разделе [Видимость](set-app-pricing-and-availability.md#visibility) на странице **Цены и доступность** выберите параметр **Отправить продукт в Магазин, но запретить его поиск там**.  Выберите параметр **Отмена возможности приобретения. Любой пользователь, у которого есть прямая ссылка, сможет увидеть описание приложения в Магазине. Но такой пользователь сможет скачать его только в двух случаях: если он ранее пользовался этим продуктом или если имеет промокод и устройство с Windows10**. Это препятствует обнаружению вашего приложения пользователями в Магазине через поиск или обзор.
-   После того как ваше приложение пройдет сертификацию, [создайте рекламные коды](generate-promotional-codes.md) приложения и распределите их между тестировщиками. Вы можете создать до 1600 активационных кодов для одного приложения в течение полугода. Эти коды предоставят вашим тестировщикам прямую ссылку на приложение в каталоге и позволят им загрузить его бесплатно, даже если вы задали для него цену во время отправки.

Если вы распространяете ссылки на рекламные коды между вашими тестировщиками, последние могут испытать приложение и предоставить вам свой отзыв, чтобы вы смогли его улучшить. Позже, когда вы решите сделать приложение доступным для всех, вы можете создать новые данные отправки и изменить значение параметра **Видимость** на **Сделать это приложение доступным и обнаруживаемым в Магазине**, а также внести любые другие изменения.

Вот некоторые моменты, которые при этом следует учитывать:

-   В любой момент вы можете дать вашим тестировщикам обновленную версию приложения, создав новые данные отправки. Убедитесь, что для параметра **Видимость** выбрано значение **Отправить продукт в Магазин, но запретить его поиск там**, если также выбран параметр **Отмена возможности приобретения**. Тестировщики получат обновление после того, как оно пройдет процедуру сертификации, однако больше никто доступ к обновлению не получит.
-   Ваши тестировщики должны использовать устройство под управлением Windows 10, на которое они смогут установить приложение. (Однако ваше приложение не обязательно должно включать пакеты Windows10 для использования этого метода тестирования.)
-   В любое время можно создать дополнительные [промокоды](generate-promotional-codes.md) для распространения (до 1600 активационных кодов на одно приложение каждые шесть месяцев).
-   Вы можете отозвать доступ к приложению после его скачивания вашими тестировщиками. После загрузки они могут продолжить использовать приложение и получать доступ к любым впоследствии опубликованным вами обновлениям.
-   Необходимо определить, каким образом вы хотите собирать отзывы ваших тестировщиков. Рассмотрите возможность предоставить ссылку в бета-версии приложения, с помощью которой ваши тестировщики смогут легко отправлять отзывы по электронной почте или через [Центр отзывов](../monetize/launch-feedback-hub-from-your-app.md).
-   Вы можете просматривать [аналитические отчеты](analytics.md) по вашему приложению, включая отчеты об использовании и работоспособности, а также любые оценки и отзывы, оставленные вашими тестировщиками.
-   Вы можете включить надстройки при предоставлении приложения тестировщикам. Так как, вероятно, вы не будете брать с них деньги, убедитесь, что для цены надстройки во время тестирования установлено значение **Бесплатно**. Затем, когда приложение станет доступно другим пользователям, вы сможете создать новую отправку для изменения цены каждой надстройки.


## <a name="other-methods-for-distributing-apps-to-testers"></a>Другие методы распространения приложений среди тестировщиков

Вы также можете ограничить область распространения приложения только целевой группой пользователей с помощью дополнительных параметров в разделе [Видимость](set-app-pricing-and-availability.md#visibility) на странице **Цены и доступность** при отправке приложения. Учтите, что эти параметры работают не во всех версиях операционных систем. В частности, ни один из них не работает в Windows8 или Windows8.1.

### <a name="targeted-distribution-to-customers-with-a-link-to-your-apps-listing"></a>Целевое распространение среди пользователей с ссылкой на ваше приложение в каталоге

При использовании этого параметра ваше приложение смогут загрузить только те люди, которые имеют прямую ссылку на ваше приложение в каталоге. Вы можете найти этот **URL-адрес** на странице [Удостоверение приложения](view-app-identity-details.md) на информационной панели. Пользователи не смогут найти приложение при поиске или просмотре позиций в Магазине, но каждый, у кого есть ссылка, сможет скачать его на устройство под управлением Windows Phone8.1 (или более ранних версий) либо Windows10. 

> [!NOTE]
> Для цены приложения должно быть выбрано значение **Бесплатно**, чтобы тестировщики могли свободно скачать его.

Чтобы воспользоваться этим параметром, выберите **Отправить продукт в Магазин, но запретить его поиск там** в разделе [Видимость](set-app-pricing-and-availability.md#visibility) на странице **Цены и доступность**. Затем выберите значение **Обязательное наличие прямых ссылок. Любой пользователь, у которого есть прямая ссылка на описание продукта, может скачать его (за исключением пользователей Windows8.x)**.  


### <a name="targeted-distribution-to-customers-with-specified-email-addresses"></a>Целевое распространение пользователям с заданными адресами электронной почты

В случае тестирования **на Windows Phone 8.1 и более ранних версий** этот параметр дает возможность ограничить распространение приложения. Ваше приложение смогут загрузить по прямой ссылке в каталоге только те пользователи, чьи адреса электронной почты (связанные с их учетными записями Майкрософт) вы укажете в этом поле.

> [!IMPORTANT]
> Пользователи с указанными вами адресами электронной почты смогут скачать приложение только на устройства с Windows Phone8.1 или более ранними версиями.
 
Вы можете найти прямую ссылку на ваше приложение на странице [Удостоверение приложения](view-app-identity-details.md) информационной панели. Пользователи не смогут найти приложение при поиске или просмотре позиций в Магазине, и даже при наличии ссылки на ваше приложение в каталоге они не смогут загрузить его, если не воспользуются учетной записью Майкрософт, связанной с электронным адресом, который вы указали при отправке этого приложения.

> [!NOTE]
Если используется этот параметр, вы все же можете сделать приложение доступным тестировщикам на устройствах с Windows10, [создав рекламные коды](generate-promotional-codes.md), как описано выше. Любой пользователь, имеющий один из рекламных кодов на ваше приложение, сможет загрузить его на устройство Windows 10, даже если вы не указали здесь его электронную почту.

Чтобы воспользоваться этим параметром, выберите **Отправить продукт в Магазин, но запретить его поиск там** в разделе [Видимость](set-app-pricing-and-availability.md#visibility) на странице **Цены и доступность**. Выберите параметр **Только для пользователей Windows Phone8.x. Только люди, которых вы укажете ниже, смогут скачать этот продукт на устройство с Windows Phone8.x. Все, у кого есть промокод и прямая ссылка, смогут скачать этот продукт на устройство с Windows10.** 

При выборе этого параметра следует учитывать следующее:

-   Этот параметр может быть выбран, если вы никогда ранее не публиковали приложение с установленным для параметра [Видимость](set-app-pricing-and-availability.md#visibility) значением **Сделать приложение доступным в Магазине**.
-   Ваше приложение должно быть оценено как **Бесплатно**, чтобы тестировщики могли свободно загрузить его.
-   Ваши тестировщики могут загрузить приложение только при использовании Windows Phone 8.1 и ниже. Тестировщики должны иметь доступное в свободной продаже устройство Windows Phone, но это устройство не обязательно должно быть разблокировано или зарегистрировано.
-   Ваши тестировщики должны иметь учетную запись Майкрософт для получения доступа к Магазину Windows и загрузки вашего приложения. Вы должны знать адрес электронной почты, связанный с учетной записью Майкрософт, каждого бета-тестера, чтобы добавить их в свой список. Для создания новой учетной записи Майкрософт тестировщики могут перейти на страницу [настройки учетной записи Майкрософт](http://go.microsoft.com/fwlink/p/?LinkId=618945).
-   Вы можете предоставить до 10000 адресов электронной почты в текстовом поле.
-   Адреса электронной почты должны быть разделены точками с запятой.
-   Вы можете добавить дополнительные адреса позже, однако для этого вам необходимо создать новую отправку.
-   Вы можете отозвать доступ к приложению после его загрузки вашими тестировщиками. После загрузки вашего приложения они смогут продолжить использовать его и получать любые предоставляемые вами обновления.
