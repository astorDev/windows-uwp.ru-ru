# Бета-тестирование и целевое распространение


Даже самое тщательное тестирование приложения не сравнится с реальной проверкой другими пользователями. Информационная панель центра разработки для Windows позволяет предоставлять доступ к приложению только для определенных людей, например, для бета-тестеров, чтобы они проверили работу приложения перед его публикацией. Бета-тестеры могут обнаружить проблемы, которые вы пропустили, например, орфографические ошибки, неясную работу приложения или ошибки, которые могут привести к сбоям приложения. У вас будет возможность исправить эти проблемы перед выпуском общедоступного приложения, и в результате вы получите более доработанный продукт.

Мы предоставляем несколько способов ограничить область распространения приложений только тестировщиками, не создавая отдельную версию приложения с другим именем и идентификатором пакета. Разумеется, если требуется, вы можете создать отдельное приложение для тестирования. В таком случае присвойте ему имя, которое отличается от окончательного публичного имени приложения.

Метод распространения приложения тестировщикам зависит от операционных систем, для которых предназначено ваше приложение. Ниже перечислены варианты, актуальные для Windows10, а также Windows Phone 8.1 и более ранних версий.

## Предоставление доступа к вашему приложению тестировщикам на устройствах Windows 10

Мы предоставляем два способа ограничить область распространения приложений только определенными пользователями с устройствами Windows 10.

### Тестовые пакеты

Если вы уже опубликовали какую-либо версию своего приложения, вы можете создавать тестовые пакеты, чтобы распространять указанным пользователям другие наборы пакетов. Вы можете создать несколько тестовых пакетов одного приложения для разных групп пользователей. Это отличный способ испытать несколько пакетов одновременно, который позволяет переносить тестируемые возможности в обычную отправку, если вы решите, что их можно публиковать для всех.

Дополнительные сведения можно найти в разделе [Тестовые пакеты](package-flights.md).

> **Примечание**. Для распространения пакетов случайной выборке пользователей Windows 10 с указанной доле, а не заданной группе пользователей, можно использовать [постепенный выпуск пакетов](gradual-package-rollout.md). Можно также объединить выпуск с тестовыми пакетами, если требуется постепенно распространять обновление в одной из тестовых групп.

### Скрытие приложения в Магазине и использование рекламных кодов

Если вам требуется ограничить область распространения приложения определенной группой тестировщиков, **не** публикуя общедоступную отправку, вы можете использовать тот же [процесс отправки приложения](app-submissions.md), что и для остальных приложений. Чтобы разрешить только определенным пользователям получать приложение бесплатно, а также скрыть описание приложения от других пользователей и запретить им скачивать его, сделайте следующее:

-   На странице **Цена и доступность** вашей отправки выберите параметр **Скрыть это приложение и запретить его покупку. Клиенты с рекламным кодом могут скачивать его на устройства под управлением Windows 10** в разделе [Распространение и видимость](set-app-pricing-and-availability.md#distribution-and-visibility). Это препятствует обнаружению вашего приложения любыми пользователями в Магазине через поиск или обзор.
-   После того, как ваше приложение пройдет сертификацию, [создайте рекламные коды](generate-promotional-codes.md) для вашего приложения для распределения между тестировщиками. Вы можете создать до 250 рекламных кодов для одного приложения в течение полугода. Эти коды предоставят вашим тестировщикам прямую ссылку на приложение в каталоге и позволят им загрузить его бесплатно, даже если вы задали для него цену во время отправки.

Если вы распространяете ссылки на рекламные коды между вашими тестировщиками, последние смогут загрузить приложение бесплатно, опробовать его и предоставить вам свой отзыв, чтобы вы смогли его улучшить. Позже, когда вы решите сделать приложение доступным для всех, вы можете создать новую отправку и изменить значение параметра **Распространение и видимость** на **Сделать приложение доступным в Магазине**, а также внести любые другие изменения.

Вот некоторые моменты, которые при этом следует учитывать:

-   В любой момент вы можете дать вашим тестировщикам обновленную версию приложения, создав новую отправку. Для параметра **Распространение и видимость** оставьте значение **Скрыть это приложение и запретить его покупку. Клиенты с рекламным кодом могут скачивать его на устройства под управлением Windows 10**. Тестировщики получат обновление после того, как оно пройдет процедуру сертификации, однако больше никто доступ к обновлению не получит.
-   Ваши тестировщики должны использовать устройство под управлением Windows 10, на которое они смогут установить приложение. (Однако ваше приложение не обязательно должно включать пакеты Windows 10 для использования этого метода тестирования.)
-   В любой время можно создать дополнительные [рекламные коды](generate-promotional-codes.md) для распространения (до 500 кодов каждые шесть месяцев).
-   Вы можете отозвать доступ к приложению после его загрузки вашими тестировщиками. После загрузки они могут продолжить использовать приложение и получать доступ к любым впоследствии опубликованным вами обновлениям.
-   Необходимо определить, каким образом вы хотите собирать отзывы ваших тестировщиков. Укажите в бета-версии адрес электронной почты или ссылку на веб-сайт, чтобы они могли без труда оставлять свои комментарии.
-   Вы можете просматривать [аналитические отчеты](analytics.md) по вашему приложению, включая любые рейтинги и отзывы, оставленные вашими тестировщиками.
-   Вы можете включить надстройки при предоставлении приложения тестировщикам. Так как, скорее всего, вы не будете брать с них деньги, убедитесь, что цена на надстройки во время тестирования установлена как "Бесплатно". Затем, когда приложение станет доступно другим пользователям, вы сможете создать новую отправку для изменения цены каждой надстройки.

## Другие методы распространения приложений среди тестировщиков

Вы также можете ограничить область распространения приложения только целевой группой пользователей с помощью дополнительных параметров в разделе [Распространение и видимость](set-app-pricing-and-availability.md#distribution-and-visibility) на странице **Цены и доступность** при отправке приложения. Учтите, что эти параметры работают не во всех операционных системах. Указанные выше параметры рекомендуется использовать при тестировании приложения на устройствах с Windows 10.

Если вы выберите любой из приведенных выше параметров, то всегда можете отправить обновление и установить для параметра [Распространение и видимость](set-app-pricing-and-availability.md#distribution-and-visibility) значение **Сделать приложение доступным в Магазине**, когда будете готовы завершить период тестирования и сделать свое приложение доступным всем пользователям. Вам не обязательно следует изменять имя приложения и создавать полностью отдельное приложение (если, конечно, это не то, чего вы добиваетесь).

### Целевое распространение среди пользователей с ссылкой на ваше приложение в каталоге

При использовании этого параметра ваше приложение смогут загрузить только те люди, которые имеют прямую ссылку на ваше приложение в каталоге. Вы можете найти этот **URL-адрес** на странице [Удостоверение приложения](view-app-identity-details.md) на информационной панели. Пользователи не смогут найти это приложение с помощью поиска или просмотра Магазина, но любой пользователь, имеющий ссылку, сможет загрузить данное приложение. (Обратите внимание, что ваше приложение должно быть оценено как **Бесплатно**, чтобы тестировщики могли свободно загрузить его.)

Чтобы использовать этот параметр, выберите **Скрыть приложение в Магазине. Пользователи, у которых есть прямая ссылка на приложение, все равно смогут его скачать (кроме пользователей ОС Windows 8 и Windows 8.1):** в разделе [Распространение и видимость](set-app-pricing-and-availability.md#distribution-and-visibility) страницы **Цены и доступность** при отправке приложения.

> **Важно**. Этот параметр не работает для тестировщиков на устройствах под управлением Windows 8 или Windows 8.1.

### Целевое распространение пользователям с заданными адресами электронной почты

В случае тестирования **на Windows Phone 8.1 и более ранних версий** этот параметр дает возможность ограничить распространение приложения. Ваше приложение смогут загрузить по прямой ссылке в каталоге только те пользователи, чьи адреса электронной почты (связанные с их учетными записями Майкрософт) вы укажете в этом поле.

> **Важно**. Пользователи с указанными вами адресами электронной почты смогут скачать приложение только на устройства с Windows Phone 8.1 или более ранними версиями.
 
Вы можете найти прямую ссылку на ваше приложение на странице [Удостоверение приложения](view-app-identity-details.md) информационной панели. Пользователи не смогут найти приложение при поиске или просмотре позиций в Магазине, и даже при наличии ссылки на ваше приложение в каталоге они не смогут загрузить его, если не воспользуются учетной записью Майкрософт, связанной с электронным адресом, который вы указали при отправке этого приложения.

> **Примечание**. Если используется этот параметр, вы все еще можете сделать приложение доступным тестировщикам на устройствах Windows 10, [создав рекламные коды](generate-promotional-codes.md), как описано выше. Любой пользователь, имеющий один из рекламных кодов на ваше приложение, сможет загрузить его на устройство Windows 10, даже если вы не указали здесь его электронную почту.

Чтобы использовать этот вариант, при отправке приложения выберите пункт **Скрыть это приложение и оставить его доступным только для указанных ниже пользователей, которые могут скачивать это приложение на устройства с WindowsPhone8.x. Для скачивания этого приложения на устройства с Windows10 можно использовать рекламный код** в разделе [Распространение и видимость](set-app-pricing-and-availability.md#distribution-and-visibility) на странице **Цены и доступность**.

При выборе этого параметра следует учитывать следующее:

-   Этот параметр может быть выбран, если вы никогда ранее не публиковали приложение с установленным для параметра [Распространение и видимость](set-app-pricing-and-availability.md#distribution-and-visibility) значением **Сделать приложение доступным в Магазине**.
-   Ваше приложение должно быть оценено как **Бесплатно**, чтобы тестировщики могли свободно загрузить его.
-   Ваши тестировщики могут загрузить приложение только при использовании Windows Phone 8.1 и ниже. Тестировщики должны иметь доступное в свободной продаже устройство Windows Phone, но это устройство не обязательно должно быть разблокировано или зарегистрировано.
-   Ваши тестировщики должны иметь учетную запись Майкрософт для получения доступа к Магазину Windows и загрузки вашего приложения. Вы должны знать адрес электронной почты, связанный с учетной записью Майкрософт, каждого бета-тестера, чтобы добавить их в свой список. Для создания новой учетной записи Майкрософт тестировщики могут перейти на страницу [настройки учетной записи Майкрософт](http://go.microsoft.com/fwlink/p/?LinkId=618945).
-   Вы можете предоставить до 10000 адресов электронной почты в текстовом поле.
-   Адреса электронной почты должны быть разделены точками с запятой.
-   Вы можете добавить дополнительные адреса позже, однако для этого вам необходимо создать новую отправку.
-   Вы можете отозвать доступ к приложению после его загрузки вашими тестировщиками. После загрузки вашего приложения они смогут продолжить использовать его и получать любые предоставляемые вами обновления.
