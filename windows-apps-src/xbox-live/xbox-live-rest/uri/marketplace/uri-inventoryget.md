# <a name="get-usersmeinventory"></a>GET (/users/me/inventory)
Предоставляет набор инвентаризации, сопоставленные указанный пользователь обратно вызывающему.
Доменом для таких URI является `inventory.xboxlive.com`.

  * ["Примечания"](#ID4EV)
  * [Параметры строки запроса](#ID4EHB)
  * [Пример запроса](#ID4EDE)
  * [Текст ответа](#ID4ERE)

<a id="ID4EV"></a>


## <a name="remarks"></a>Замечания

Политика не проверяет, в случае принудительного применения или фильтрация будет выполняться как часть этого вызова. Можно передать параметры запроса, чтобы сузить перечень результатов, возвращаемых у вызывающих объектов.

Вызывающие объекты могут постраничного просмотра результатов с маркером продолжения как предоставляются через предыдущего ответа: **/users/me/inventory? continuationToken = continuationTokenString**.

Вызывающий объект может делать вызов к сведениям о API с помощью URL-адрес конкретного элемента для просмотра сведений о конкретном элементе.

<a id="ID4EHB"></a>


## <a name="query-string-parameters"></a>Параметры строки запроса

| Параметр| Тип| Описание|
| --- | --- | --- |
| доступность| Строка| Текущий доступность возвращаемых элементов. Значение по умолчанию — «Доступна», который возвращает элементы, для которых текущая дата находится между датой начала и конца диапазона дат. Другие значения: «Все», который возвращает все элементы и «Недоступно», который возвращает элементы, для которых текущая дата выходит за пределы диапазона дат начала даты и окончания и его таким образом в данный момент недоступно. |
| Контейнер| Строка| Необязательно. Если значение равно идентификатор продукта игры, затем результаты из данных инвентаризации включают только элементы, связанные с этой игре. Это особенно полезно при вызове инвентаризации с сервера для фильтрации результатов до определенную игру продуктов.|
| expandSatisfyingEntitlements| Строка| Флаг, указывающий, если ответ включает в себя все удовлетворителен права, которые пользователь в результаты возвращены. Значение по умолчанию — «false». Если этот параметр используется со значением «true», все продукты, которые предоставляются пользователю через удовлетворяющие прав таких как объединенные элементы, Xbox 360 покупки перенесены на Xbox One, на преимущества подписки, и т.д. добавляются к результатам. Если это значение равно «false» только родительские элементы например ProductID пакета возвращаются в результаты, а не отдельных включенных элементов. **Примечание.** С помощью этого параметра со значением «true» поддерживается только если параметр itemType не включен в URI, в противном случае появится сообщение об ошибке HTTP 400. |  
  | productIds | Строка |  Коллекция идентификаторов ProductID, необходимо специально получить из данных инвентаризации пользователя, разделенных точкой «,».  Если пользователь не имеет указанный ProductID в результатах инвентаризации, этот элемент не появится в результатах из вызова API. Если передать productID в пакете, а также набор параметров expandSatisfyingEntitlements значение true, все элементы, включенные в пакет возвращаются результаты вызова (ли их идентификаторов ProductID указан в строке запроса или нет).   |
  | Состояние | Строка | Состояние возвращаемых элементов. Значение по умолчанию — «все», который возвращает все элементы. Другие значения, уже включена, указывает, что только itemsthat включены должны быть возвращены, «Приостановлен», которое указывает, что только элементы, которые будут приостановлены должны быть возвращены, «Срок действия истек», который указывает, что должно возвращаться только те элементы, которые срок действия истек,» Отменено», который указывает, что должны быть возвращены только элементы, которые будут отменены, и «Renewed», которое указывает, что должны быть возвращены только элементы, которые были обновлены.  |

Помимо этого ресурс поддерживает стандартный механизм разбиения на страницы.

## <a name="sample-request"></a>Пример запроса

— Полное доменное имя для этого метода URI `https://inventory.xboxlive.com/users/me/inventory.
         `

> [!NOTE] 
> Пользователей, которые считаются зависит от токеном, который может включать несколько пользователей. Если требуется Инвентаризация одного пользователя, необходимо также указать хэш-код пользователя для конкретного пользователя, который следует рассматривать исключительно.
