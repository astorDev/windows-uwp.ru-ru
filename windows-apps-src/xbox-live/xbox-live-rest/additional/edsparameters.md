# <a name="eds-parameters"></a>Параметры EDS

<a id="ID4EO"></a>


## <a name="query-string-parameters"></a>Параметры строки запроса

Эти параметры запросов, не принимаются обязательно все [развлечения обнаружения службы (EDS) API-интерфейсы](../uri/marketplace/atoc-reference-marketplace.md), но все принимает более одного API.

| Параметр| Тип| Описание|
| --- | --- | --- |
| combinedContentRating| Строка| Необязательно. См. в разделе [получить (/media/ {marketplaceId} / contentRating)](../uri/marketplace/uri-medialocalecontentratingget.md).|
| continuationToken| Строка| Необязательно. Токен продолжения — это непрозрачный BLOB-объект, содержащий сведения, служба должна для разбиения на страницы в определенных сценариях. Если значение не задано, первая страница результатов возвращается (где размер страницы определяется параметром maxItems), а также маркер продолжения, который может использоваться для получения второй страницы результатов. Вторая страница будет содержать токен продолжения на третьей странице результатов и так далее.|
| требуемый| Строка| Необязательно. См. в разделе API Name объединенного поля.|
| desiredMediaItemTypes| Массив строк| Необязательно. Этот параметр определяет типы элементов, возвращаемых в ответе.|
| Домен| Строка| Необязательно. Параметр домена определяет контекст marketplace игр и приложений в вызывает для клиента. По умолчанию доменом является «Современный», указывающий, что клиент может запрашивать только для Xbox One содержимого. Если клиент хочет переключить домен на поверхности содержимое Xbox 360, его необходимо указать домен как «Xbox360». В настоящее время междоменная результатов не поддерживаются. Возможные значения: <ul><li>Xbox360</li><li>Современные</li></ul> Значение «Xbox360» поддерживается только для singleMediaGroupSearch. Обзор и сведения о поддерживаются. crossMediaGroupSearch не поддерживается и возвращает ошибку 400.|
| поля| Строка| Необязательно. См. в разделе [получить (/media/ {marketplaceId} / поля)](../uri/marketplace/uri-medialocalefieldsget.md).|
| firstPartyOnly| Логическое значение| Параметр дополнительный фильтр. Определяет, возвращается только содержимое первой или ли оба первый - и стороннего содержимого возвращенные из запроса. |
| freeOnly| Логическое значение| Параметр дополнительный фильтр. Ограничивает результаты только бесплатные содержимого.|
| GroupBy| TK| Чтобы классифицировать результирующий набор на группы, а не один результирующий набор используется параметр groupBy. При указании этого параметра будет изменение результирующего набора для возврата нескольких списков элементов, где число элементов в каждом сегменте определяется параметр maxItems. <ul><li>MediaGroup - результаты группируются по MediaGroup.</li></ul> |
| hasTrailer| Логическое значение| Параметр дополнительный фильтр. Определяет ли возвращаемых элементов должен содержать трейлера или если наличие трейлера является необязательным. Если значение равно true, все элементы должны иметь прицепов.|
| id| Строка| Необязательно. Если указано, ограничивает результаты только как дочерние элементы элемента с заданным идентификатором. Если этот параметр, необходимо также указать MediaItemType. |
| идентификаторы| Массив строк| Обязательный. Все идентификаторы (до 10), для которых возвращаются сведения. Обратите внимание, что любой идентификатор, который содержит символы, недопустимые для размещения URL-адрес (идентификаторы ProviderContentId типов, обычно полные URL-адреса, сами и поэтому содержат недопустимые символы) должен быть URL-адреса отправки должным образом на EDS.|
| Тип_идентификатора| Строка| Необязательно. Тип идентификаторы, которые передаются в параметре идентификаторы. Допустимые значения: <ul><li>Каноническое (Bing или Marketplace)</li><li>XboxHexTitle (воспроизведение на консоли приложение)</li></ul>  Все входящие идентификаторы должны совместно использовать же Тип_идентификатора. Если это значение указано, все идентификаторы считаются Canonical.|
| latestOnly| Логическое значение| Параметр дополнительный фильтр. Ограничивает результаты только те с последней дате выпуска.|
| maxItems| 32-разрядное знаковое целое число| Необязательно. Определяет максимальное количество элементов, которые должны быть возвращены из вызова. Допустимые значения: числа в диапазоне от 1 до 25, включительно. Параметр по умолчанию до 25, если не указано.|
| mediaGroup| Строка| Необязательно. Группа мультимедиа идентификаторов. Все входящие идентификаторы должны совместно использовать одну и ту же группу мультимедиа.|
| MediaItemType| Строка| Необязательно. Тип элемента, идентификатор которого был указан в параметр id. Если указан параметр id, необходимо также указать этот параметр.|
| OrderBy| Строка| Обязательный. Параметр orderBy определяет порядок сортировки, возвращаемых элементов. Здесь перечислены общие значения для этого поля, но некоторые интерфейсы API могут поддерживать дополнительные значения.<ul><li>playCountDaily - по количеству раз воспроизводить мультимедиа, для последнего дня.</li><li>freeAndPaidCountDaily - по количеству приобретений бесплатные и платные, для последнего дня.</li><li>paidCountAllTime - по числу только платных покупки, за весь период.</li><li>paidCountDaily - по количеству платных покупок, для последнего дня.</li><li>digitalReleaseDate - по дате для загрузки.</li><li>releaseDate - что по дате в хранилищах, откат к дате выпуска цифровой (если доступно).</li><li>userRatings - по среднему пользователю оценки.</li></ul> |
| preferredProvider| Строка| Необязательно. Если пользователь имеет предпочтительного поставщика содержимого, например Comcast Xfinity или Verizon FIOS, его идентификатор можно передать в. Фактический порядок элементов не изменится, для каждого элемента сведения указанного поставщика будет отображаться в верхней части списка поставщиков (Если предпочтительный поставщика содержимого элемента доступны).|
| q| Строка| Обязательный. Запрос термин, используемый в службе поиска.|
| queryRefiners| Массив строк| Необязательно. См. в списке [уточнения запроса EDS](edsqueryrefiners.md).|
| связь| Строка| Необязательно. Фильтр, который использует параметр ID как основу для поиска для других продуктов, которые соответствуют указанный тип связи: <ul><li>bundledWith - поиск пакета продукты, где параметр ID входит в состав этих пакетов.</li><li>bundledProducts - найти продукты, включенные в пакет, указанный параметром ID.</li></ul>  С помощью этого параметра будет возвращаться только продукты, видимые в marketplace (могут быть возвращены в вызов обзора). Если пакет имеет скрытым продуктом, он по-прежнему является частью пакета, но в этих результатах.|
  | ScopeId | Строка | Этот параметр используется в сценариях обратного просмотра для мультимедиа. |
  | ScopeIdType | Строка | Этот параметр используется в сценариях обратного просмотра для мультимедиа. Возможные значения: Заголовок. |
  | skipItems | 32-разрядное знаковое целое число | Необязательно. Для разбиения на страницы в сценариях между группами skipItems параметр используется для определения, сколько элементов, уже видели (и тем самым какой элемент должен быть отображен сначала в результирующем наборе). Значение отсчитывается от 0, поэтому skipItems = 0 (или просто не указываются skipItems) начинается извлечение в начале списка. skipItems = 3 бы пропустить первые три элемента в списке и начать извлечение с четвертого элемента. |
  | subscriptionLevel | Массив строк | Параметр дополнительный фильтр. Параметр subscriptionLevel определяет, что тип подписки пользователя (например, имеет ли пользователь платную или бесплатную подписку). Ниже приведены возможные значения. <ul><li>Gold: Пользователь есть платная подписка</li><li>Silver: Пользователь имеет бесплатную подписку.</li></ul>  |
| Оконечнымустройствам| Строка| EDS предоставляет возможность фильтрации предлагает для целевых устройств. Предложений (ProviderContent или доступность), возвращаемых для элемента может быть ограничен на целевое устройство.|
| topRatedOnly| Логическое значение| Параметр дополнительный фильтр. Ограничивает результаты только наиболее номинальной содержимое.|
