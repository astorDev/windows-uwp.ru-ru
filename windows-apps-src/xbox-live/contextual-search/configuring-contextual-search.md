# <a name="configuring-contextual-search"></a>Настройка контекстного поиска

## <a name="configuration-info"></a>Сведения о конфигурации

### <a name="designing-your-data"></a>Проектирование данных
При настройке заголовка для контекстного поиска, важно думать о том, которые делают уникальное название.  Если пользователь ищет видеосодержимого, что может быть сталкивается для поиска?  Насколько детальным данные должны быть?

Контекстного поиска поддерживает возможность фильтрации и сортировки содержимого, метаданные должно отражать эти конструкции поиска для охвата отлично.  Хорошее проверенное правила, представьте себе игровых данных в двух сводки:
1. Игр состояние - ex.  Уровень, символ, оружия, карты, роли, точка контрольной точки или историю, DLC
2. Квалификации пользователя - ex. Соотношение K/D, престиж, ранг, скорость выполнения

Первый pivot отлично подходит для фильтрации содержимого, второй — отлично подходит для сортировки.  Кроме того в зависимости от типа содержимого, отображаемые, различных уровнях детализации stat важны.  Для широковещательных рассылок, при условии, что имеется ограниченное число активных широковещательные рассылки, в любой момент времени ниже гранулярности данных обеспечит с большей вероятностью успешного поиска.  Для игр клипы постоянно возрастает, поэтому большей гранулярности данных обеспечит более точные результаты.

Это часто случается, что у вас уже есть эти данные, созданные функции, Xbox Live, как присутствие Rich или Stats Hero, так что это отличная возможность воспользоваться уже завершили работу.

Если вы используете статистики для фильтрации содержимого, имейте в виду, что статистики всех контекстного поиска доступны для запроса в любое время.  Для этого следует разрабатывать вашей регистрации событий соответствующим образом.

Например статистика, например SinglePlayerMap и MultiplayerMap при использовании для фильтрации содержимого, проигрыватель будет только одновременно находиться в одном из них.  Тем не менее оба значения будут доступны для запросов от службы в любое время.  Очень важно, как вы задали снять другие.  Для строки на основе статистики, отлично подходит пустая строка (Убедитесь, что не нужно включать, в вашей конфигурации пользовательского интерфейса, при необходимости).

### <a name="configuring-a-stat-for-contextual-search"></a>Настройка в состоянии для контекстного поиска
Настройка заголовка для контекстного поиска несложно, после настройки событий и статистики эту степень тегов.  См. другие имеющейся документации XDP или центра партнеров по настройке контекстного поиска, если вы еще не знакомы.

![](../images/contextual_search/config02.png)

После вы нажмете плитку контекстного поиска в основной службы конфигурации части XDP, приведенном выше рисунке приведен пример на главной странице.

Даже если это первый раз, которые просматривали эту страницу, это возможно, что вы увидите stats, уже настроен для контекстного поиска.  Это обусловлено статистики всех вашего героя автоматически настраиваются для контекстного поиска. Вам не нужно хранить эти статистические данные Hero в конфигурации, но они обычно прекрасно подходит для сортировки (и уже готово для вас).

В настоящее время максимальное число stat экземпляров, которые можно настроить для контекстного поиска — 10.

На этой странице каждый из статистики представлен следующим образом: Приоритет — отображаемое имя (имя Stat экземпляра)

Каждый из них будет рассматриваться более подробно в следующем разделе.

![](../images/contextual_search/config01.png)

Приведенном выше рисунке весьма экран, который будет отображаться, если вы решили создать новый контекстного поиска stat или изменить существующий.

Чтобы успешно настроить stat для контекстного поиска, выполните следующие действия:
1. Выберите свой экземпляр stat.

  ![](../images/contextual_search/config03.png)

  Обратите внимание, что stat только экземпляры поддерживаются - stat шаблоны не принимаются.  Также следует помнить о видимости, которые заданы для stat экземпляра (настраивается в части Stats XDP или центра партнеров).  Только статистика, помеченные как **откройте** будет отображаться в возможности третьих лиц.

2. Выберите stat приоритет. Это способ разделения важность Эта статистика по отношению к другим для поиска взаимодействия и алгоритмы.  Допустимые значения от 1 до 10 (1 — самым высоким).  Оставьте как 0 или пустым для этого нужно игнорировать.
3. Добавьте отображаемое имя.  Это является Локализованная строка, которая предоставляется конечному пользователю.
4. Проверьте, будет ли использовать Эта статистика для фильтрации или сортировки результатов.  Вы можете проверить, как в некоторых случаях (если значение stat является числом - желательно в определенный диапазон).
5. Выберите представление stat значение.  В случае у вас есть три варианта.
   * Необработанное значение - stat представляется как есть и не имеет диапазон требований.  Это лучше всего подходит для сортировки.
   * SET - значения сопоставляются с отдельными локализуемые строки.  Это лучше всего использовать для фильтрации.
   * Диапазон - значения, которые попадают в min и max диапазона.  Этот тип является отлично подходит, когда нужно выполнить фильтрацию и сортировку на основе этих данных.

#### <a name="explaining-sets"></a>Объясняя наборов
Если вы не знакомы с наборами, они предназначены для сопоставления потенциальные значения stat локализуемые строки, которые могут быть предоставлены для конечного пользователя.  Они очень важны для контекстного поиска статистики, который вы хотите отфильтровать с помощью.

### <a name="updating-your-contextual-search-document"></a>Обновление документа контекстного поиска
В настоящее время можно легко изменить stats контекстного поиска документа, как мы только поддерживают передачу данных в данный момент (который представляют собой временные данные).  Как только мы начинаем для поддержки игр клип, добавление тегов (сентябрь 2015 г.), если вы решили добавить или удалить статистики, будут влиять клипы, которые уже были индексированы.  При добавлении нового stat только игр клипов и широковещательные рассылки, созданные после изменения вступают в силу с тегами и отображается в запросах с использованием нового stat. Старые игры клипов не смогут запроса-из Эта статистика. При удалении stat из документа контекстного поиска старого stat, подключенный к игр клип будет бесполезной и не будут возвращены.
