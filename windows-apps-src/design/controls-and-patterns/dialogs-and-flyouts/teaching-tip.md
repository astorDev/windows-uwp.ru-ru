# <a name="teaching-tip"></a>Совет обучения

Совет саду является частично постоянные и мультимедийного всплывающего меню, предоставляющий контекстную информацию. Он часто используется для информацией о том, о том и обучение пользователей о важных и новых функций, которые могут улучшить впечатления пользователей.

**Важные API:** [Класс TeachingTip](https://docs.microsoft.com/en-us/uwp/api/microsoft.ui.xaml.controls.teachingtip)

Совет обучения может быть отклонить light или требуют явного действия, чтобы закрыть окно. Совет саду предназначенных для конкретного элемента пользовательского интерфейса с его заключительного фрагмента, а также использоваться без заключительного фрагмента или целевой.

## <a name="is-this-the-right-control"></a>Выбор правильного элемента управления 

Используйте **TeachingTip** элемента управления, чтобы уделить внимание пользователя новых и важных обновлений и функций, Напоминать пользователю необязательных параметров, которые бы улучшить впечатления пользователей или провести обучение пользователя, как задача должна быть выполнена. 

Поскольку tip саду является временной, не было бы рекомендуемые элементы управления для запроса пользователей об изменении важные состояния или ошибки.

### <a name="create-a-teaching-tip"></a>Создать совет обучения

Ниже приведен XAML для целевых саду элемента управления, который демонстрирует внешний вид по умолчанию TeachingTip с заголовок и подзаголовок подсказки. Обратите внимание, что саду tip может находиться в любом в дереве элементов или кода программной части. В этом примере он расположен в ResourceDictionary.

XAML
```XAML
<Button x:Name="SaveButton" Content="Save">
    <Button.Resources>
        <controls:TeachingTip x:Name="AutoSaveTip"
            Target="{x:Bind SaveButton}"
            Title="Save automatically"
            Subtitle="When you save your file to OneDrive, we save your changes as you go - so you never have to.">
        </controls:TeachingTip>
    </Button.Resources>
</Button>
```

C#
```C#
public MainPage()
{
    this.InitializeComponent();

    if(!HaveExplainedAutoSave())
    {
        AutoSaveTip.IsOpen = true;
        SetHaveExplainedAutoSave();
    }
}
```

### <a name="non-targeted-tips"></a>Целевые не советы

Не все советы относятся к элементу на экране. В таких случаях не задано целевое свойство и подсказки саду вместо отображают относительно краев корневого элемента xaml. Тем не менее ассистенты совет может иметь заключительного удалены сохранив размещения по отношению к элементу пользовательского интерфейса путем установки свойства TailVisibility «Свернуть». Ниже приведен подсказки не предназначенных для обучения.

XAML
```XAML
<Button x:Name="SaveButton" Content="Save" />

<controls:TeachingTip x:Name="AutoSaveTip"
    Title="Saving automatically"
    Subtitle="We save your changes as you go - so you never have to.">
</controls:TeachingTip>
```

Обратите внимание, что в этом примере TeachingTip в дереве элементов, а не в ResourceDictionary или в коде программной части. Это не влияет на поведение; TeachingTip только отображается при открытии, а не место макета.

### <a name="preferred-placement"></a>Предпочтительное место размещения

Совет саду реплицирует всплывающего меню [FlyoutPlacementMode](https://docs.microsoft.com/uwp/api/Windows.UI.Xaml.Controls.Primitives.FlyoutPlacementMode) поведение при размещении в свойстве TeachingTipPlacementMode. Режим размещения по умолчанию попробует поместить совет целевых саду над целевым объектом и не предназначенных для обучения совет по центру в нижней части корневого xaml. Как с помощью всплывающего меню, если в режиме предпочтительное место размещения не останется место для обучения подсказку, которую нужно отобразить, другой режим размещения будет автоматически выбран. 

Приложения, которые прогнозируют игровой входных данных, см. в разделе [gamepad и удаленное управление взаимодействия]( https://docs.microsoft.com/en-us/windows/uwp/design/input/gamepad-and-remote-interactions#xy-focus-navigation-and-interaction). Мы рекомендуем проверить доступность игровой каждый совет обучения, используя все возможные конфигурации пользовательского интерфейса приложения.

Целевой саду подсказка с его PreferredPlacement, равным «Слева снизу» отображается с окончанием, по центру в нижней части своей цели с текстом всплывающей подсказки саду сдвинуты налево.

XAML
```XAML
<Button x:Name="SaveButton" Content="Save">
    <Button.Resources>
        <controls:TeachingTip x:Name="AutoSaveTip"
            Target="{x:Bind SaveButton}"
            Title="Saving automatically"
            Subtitle="We save your changes as you go - so you never have to."
            PreferredPlacement="BottomLeft">
        </controls:TeachingTip>
    </Button.Resources>
</Button>
```

### <a name="add-a-placement-margin"></a>Добавление полей размещения  

Вы можете управлять, насколько далеко совет целевых саду задается помимо своей цели и насколько совет не предназначенных для обучения имеет значение от края корневого элемента xaml с помощью свойства PlacementMargin. Как и [Margin](https://docs.microsoft.com/en-us/uwp/api/windows.ui.xaml.frameworkelement.margin), PlacementMargin имеет четыре значения — слева, справа, сверху и снизу — только используются соответствующие значения. Например PlacementMargin.Left применяется, когда tip остается целевого объекта, или у левого края корневого элемента xaml.

В примере показан совет не предназначенном с PlacementMargin влево/верх или право/низ значение 80.

XAML
```XAML
<Button x:Name="SaveButton" Content="Save" />

<controls:TeachingTip x:Name="AutoSaveTip"
    Title="Saving automatically"
    Subtitle="We save your changes as you go - so you never have to."
    PreferredPlacement="BottomLeft"
    PlacementMargin="80">
</controls:TeachingTip>
```

### <a name="add-content"></a>Добавление содержимого

Содержимое можно добавить к совету обучения, используя свойство Content. Если имеется больше содержимого, чтобы показать, чем размер чаевых обучения, которое будет разрешать, полоса прокрутки будет включено автоматически позволяет пользователю выполнять прокрутку области содержимого. 

XAML
```XAML
<Button x:Name="SaveButton" Content="Save">
    <Button.Resources>
        <controls:TeachingTip x:Name="AutoSaveTip"
            Target="{x:Bind SaveButton}"
            Title="Saving automatically"
            Subtitle="We save your changes as you go - so you never have to.">
                <StackPanel>
                    <CheckBox x:Name="HideTipsCheckBox" Content="Don't show tips at start up" IsChecked="{x:Bind HidingTips, Mode=TwoWay}" />
                    <TextBlock>You can change your tip preferences in <Hyperlink NavigateUri="app:/item/SettingsPage">Settings</Hyperlink> if you change your mind.</TextBlock>
                </StackPanel>
        </controls:TeachingTip>
    </Button.Resources>
</Button>
```

### <a name="add-buttons"></a>Добавление кнопок

По умолчанию standard «X» кнопка "Закрыть" отображается рядом с заголовком подсказки обучения. Кнопка «Закрыть» можно настроить со свойством CloseButtonContent, в котором регистр кнопки перемещаются в нижнюю часть саду tip.

**Примечание. Нет кнопка "Закрыть" будут отображаться на свет отклонить включены советы**

Можно добавить кнопки настраиваемого действия, задав свойство ActionButtonContent (и при необходимости ActionButtonCommand и свойства ActionButtonCommandParameter).

XAML
```XAML
<Button x:Name="SaveButton" Content="Save">
    <Button.Resources> 
        <controls:TeachingTip x:Name="AutoSaveTip"
            Target="{x:Bind SaveButton}"
            Title="Saving automatically"
            Subtitle="We save your changes as you go - so you never have to."
            ActionButtonContent="Disable"
            ActionButtonCommand="DisableAutoSave"
            CloseButtonContent="Got it!">
                <StackPanel>
                    <CheckBox x:Name="HideTipsCheckBox" Content="Don't show tips at start up" IsChecked="{x:Bind HidingTips, Mode=TwoWay}" />
                    <TextBlock>You can change your tip preferences in <Hyperlink NavigateUri="app:/item/SettingsPage">Settings</Hyperlink> if you change your mind.</TextBlock>
                </StackPanel>
        </controls:TeachingTip>
    </Button.Resources>
</Button>
```

### <a name="enable-light-dismiss"></a>Закрыть свет Enable

Закрыть свет функция отключена по умолчанию, но его можно включить, чтобы совет саду приводит к отмене, например, когда пользователь прокручивает или взаимодействует с другими элементами приложения. Кнопка «Закрыть» будут автоматически удалены из совет поддержкой саду light-dismiss для определения его закрыть свет поведение для пользователей.

### <a name="escaping-the-xaml-root-bounds"></a>Экранирование границы корневой xaml

В Windows 19H версии 1 и более поздних версий, ассистенты tip можно переходить через границы корень xaml и экрана, задав свойство ShouldConstrainToRootBounds. Если это свойство включено, совет саду не будет оставаться в пределах корневой xaml, ни экрана, всегда будет позиция, в наборе PreferredPlacement режим. Мы рекомендуем включить свойство IsLightDismissEnabled и задайте режим PreferredPlacement ближайший к центру корень xaml, чтобы обеспечить максимальное удобство пользователей.

В более ранних версиях Windows это свойство учитывается и саду совет всегда остается в пределах границ корневого элемента xaml.

XAML
```XAML
<Button x:Name="SaveButton" Content="Save" />

<controls:TeachingTip x:Name="AutoSaveTip"
    Title="Saving automatically"
    Subtitle="We save your changes as you go - so you never have to."
    PreferredPlacement="BottomRight"
    PlacementMargin="-80,-50,0,0"
    ShouldConstrainToRootBounds="False">
</controls:TeachingTip>
```

### <a name="canceling-and-deferring-close"></a>Отмена важных областях и закрыть

Событие Closing можно отменить и/или отложить закрытии подсказки обучения. Это можно использовать для обучения tip не закрывайте или дождаться действие или пользовательские анимации, чтобы. При отмене закрытия совет саду IsOpen вернется значение true, тем не менее, он будет оставаться false во время отсрочки. Можно также отменить программный закрытие. 

**Примечание. Если параметр не размещения позволит совет обучения, чтобы полностью Показать, tip саду будет использовать перебор жизненного цикла события для принудительное закрытие вместо отображения без доступны кнопка "Закрыть". Если приложение отменяет событие Closing, tip саду может оставаться открытым при отсутствии доступных кнопка "Закрыть".**

XAML
```XAML
<controls:TeachingTip x:Name="EnableNewSettingsTip"
    Title="New ways to protect your privacy!"
    Subtitle="Please close this tip and review our updated privacy policy and privacy settings."
    Closing="OnTipClosing">
</controls:TeachingTip>
```

C#
```C#
public void OnTipClosing(object sender, TeachingTipClosingEventArgs args)
{
    if (args.Reason == TeachingTipCloseReason.CloseButton)
    {
        using(args.GetDeferral())
        {
            bool success = await UpdateUserSettings(User thisUsersID);
            if(!success)
            {
                //We were not able to update the settings! Don't close the tip and display the reason why.
                args.Cancel = true;
                ShowLastErrorMessage();
            }
        }
    }
}
```

### <a name="recommendations"></a>Рекомендации
* Советы по impermanent и не должны содержать информацию и параметры, которые важны для работы приложения. 
* Старайтесь избегать отображение советы саду слишком часто. Советы преподавательской деятельности, скорее всего, для каждого получения отдельного внимания, когда они по ширине, на протяжении длительного времени или в нескольких сеансах.    
* Поддерживайте лаконичный советы и очистить их разделе. Исследование показало, пользователи, в среднем, только чтение 3 – 5 слов и только воспринимать 2 – 3 слов перед тем, как взаимодействовать с чаевых.
* Доступность игровой саду совет не гарантируется. Приложения, которые прогнозируют игровой входных данных, см. в разделе [gamepad и удаленное управление взаимодействия]( https://docs.microsoft.com/en-us/windows/uwp/design/input/gamepad-and-remote-interactions#xy-focus-navigation-and-interaction). Мы рекомендуем проверить доступность игровой каждый совет обучения, используя все возможные конфигурации пользовательского интерфейса приложения.
* При включении саду Совет для экранирования корень xaml, мы рекомендуем также включить свойство IsLightDismissEnabled и задайте режим PreferredPlacement ближайший к center корневого элемента xaml.
