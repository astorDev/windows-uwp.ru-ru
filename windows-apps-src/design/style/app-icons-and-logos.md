# <a name="app-icons-and-logos"></a>Значки и логотипы приложений 

Каждое приложение содержит значок и логотип, который ее представляет и такой же значок появляется в нескольких местах в оболочке Windows: 

:::row:::
    :::column:::
        * Заголовок окна приложения
        * Список приложений, в меню "Пуск"
        * Диспетчер задач и задач
        * Плитки приложения
        * Экран-заставка приложения
        * В Microsoft Store
    :::column-end:::
    :::column:::
        ![windows 10 start and tiles](images/assetguidance01.jpg)
    :::column-end:::
:::row-end:::

В этой статье рассматриваются основы создания значков приложений, как использовать Visual Studio для управления ими и как управлять их вручную, вам нужно.
 
(Эта статья предназначена специально для значков, которые представляют самого приложения; значок Общие рекомендации, см. в разделе [значки](icons.md) статьи.)

Это очень значков! К счастью Visual Studio предоставляет средство, которое позволяет легко создавать и обновлять эти значки.

## <a name="generating-all-assets-at-once"></a>За один раз создания всех ресурсов

Первое меню элемента в **визуальные активы** вкладке **все визуальные ресурсы**, точно одноименное: создает каждый визуальных ресурсов, необходимую для работы приложения нажатием кнопки.

![Создать все визуальные ресурсы в Visual Studio](images/app-icons/all-visual-assets.png)

Все, что нужно сделать, — указать одного образа, и Visual Studio создать небольшой плитке, Средняя плитка, большая плитка, широкая плитка, большая плитка, значок приложения, экран-заставка и пакет логотипы для каждого коэффициент масштабирования.

Чтобы одновременно создать все ресурсы:
1. Нажмите кнопку **...**  рядом с полем **источника** поле и выберите образ, который вы хотите использовать. Если вы используете растрового изображения, убедитесь, что это по крайней мере 400, 400 пикселей, что позволяет получить точный результаты. Векторные изображения подходят лучше всего; Visual Studio позволяет использовать искусственный Интеллект (Adobe Illustrator) и PDF-файлов. 
2. (Необязательно). В **параметры отображения** раздел, укажите следующие параметры:

    а.  **Короткое имя**:  Укажите короткое имя для вашего приложения.

    б)  **Показывать имя**: Укажите, хотите ли вы для отображения на средний, ширину или большие плитки короткое имя. 

    в) **Мозаичного фона**: Укажите шестнадцатеричное значение или имя цвета для фона мозаики. Например, `#464646`. Значение по умолчанию: `transparent`.

    г. **Фон экрана приложения**: Укажите имя шестнадцатеричное значение или цвета для фона экрана приложения. 

3. Нажмите кнопку **создания**. 

Visual Studio создает файлы изображений и добавляет их в проект. Если вы хотите изменить свои активы, просто повторите процесс. 

Ресурсы значков масштабированное выполните это соглашение об именовании файлов.

*Имя файла*- масштабирование —*коэффициент масштабирования*.png

Например:

Square150x150Logo-scale-100.png, Square150x150Logo-scale-200.png, Square150x150Logo-scale-400.png

Обратите внимание на то, что Visual Studio не создает эмблему по умолчанию. Том, что ваш логотип уникален и, возможно, не должны соответствовать другие значки приложения. Дополнительные сведения см. в разделе [отображение значка уведомления для приложений универсальной платформы Windows в статье](/windows/uwp/design/shell/tiles-and-notifications/badges). 


## <a name="more-about-app-icon-assets"></a>Дополнительные сведения о ресурсы значков приложений
Visual Studio создаст все активы значок приложения, необходимые в проекте, но если вы хотите настроить их, это помогает понять, как они отличаются от других ресурсов приложения. 

Ресурс значка приложения появится в нескольких местах: на панели задач Windows, представления задач, ALT + TAB и нижний правый угол начала плиток. Так как ресурс значка приложения появится в самых разных местах, у него есть некоторые дополнительные изменения размера и plating другие ресурсы не имеют параметров: активы «целевой размер» и «без покрытия» активы. 

### <a name="target-size-app-icon-assets"></a>Ресурсы значков приложений целевого размера
Помимо размеров коэффициент стандартными масштабируемыми («Square44x44Logo.scale-400.png») мы также рекомендуем создания активов «целевой размер». Мы называем эти ресурсы целевого размера, так как они предназначены для определенных размеров, таких как 16 пикселей, а не специальным масштабом факторов, таких как 400. Ресурсы целевого размера предназначены для областей, которые не используют масштабирования системы плато:

* список переходов начального экрана (компьютеры)
* нижний угол плитки начального экрана (компьютеры)
* сочетания клавиш (компьютеры)
* панель управления (компьютеры)

### <a name="unplated-assets"></a>Без покрытия активы
По умолчанию Windows использует ресурс на основе целевой поверх цветной сигнальной по умолчанию. Если требуется, можно предоставить ресурс без покрытия на основе целевой объект. «Без покрытия» означает, что ресурс будет отображаться на прозрачном фоне. Имейте в виду, что эти ресурсы будут отображаться на различные цвета фона. 

![ресурсы без основы и с основой](images/assetguidance22.png)

* Список переходов панели задач

## <a name="customizing-asset-padding"></a>Настройка средства заполнения

По умолчанию генератор ресурсов Visual Studio применяет рекомендуемую отбивку любой образ. Если ваши образы уже содержат заполнения или полного выхода за край изображения, которые расширяют и заканчивая плитки, можно отключить эту функцию, сняв флажок **применить рекомендуемые заполнения** "флажок". 

### <a name="tile-padding-recommendations"></a>Рекомендации заполнения плитки
Если вы хотите предоставить собственные заполнения, ниже приведены наши рекомендации для плиток. 

4 размеров плитки: малый (71 x 71), medium (150 x 150), всего (310 x 150) и больших (310 x 310). 

Каждый ресурс плитки имеет такой же размер, что и плитка, на которой он размещен.

![Плитка, показывающая полного выхода за край](images/app-icons/tile-assets1.png)

Если вы не хотите значок таким образом, чтобы расширить до края плитки, прозрачных точек ресурса-контейнера можно использовать для создания заполнения. 

![плитка и основа](images/assetguidance05.png)

Если плитка маленькая, то значок должен иметь такую ширину и высоту, чтобы его размер составлял не более 66% от размера плитки:

![соотношения размеров на маленькой плитке](images/assetguidance09.png)

Если плитка средняя, то ширина значка должна составлять 66%, а высота 50% от размера плитки. Это позволит избежать наложения элементов в фирменной панели:

![соотношения размеров на средней плитке](images/assetguidance10.png)

Если плитка широкая, то ширина значка должна составлять 66%, а высота 50% от размера плитки. Это позволит избежать наложения элементов в фирменной панели:

![соотношения размеров на широкой плитке](images/assetguidance11.png)

Для крупных плиток ширина значка должна составлять 66 %, а высота — 50 % от размера плитки.

![соотношения размеров на широкой плитке](images/assetguidance12.png)

Некоторые значки ориентированы горизонтально или вертикально, другие имеют более сложную форму, которая не позволяет расположить их на квадратном участке в нужном размере. Значки, которые выглядят выровненными по центру, могут быть больше с одной стороны. В этом случае часть значка может выступать за контур рекомендуемого участка размещения при условии, что визуально этот значок занимает такое же пространство, как и квадратный значок:

![три выровненных по центру значка](images/assetguidance13.png)

В случае если используются полные ресурсы без заполнения, необходимо учитывать элементы, пересекающие поля и края плиток. Поля должны занимать не менее 16 % высоты или ширины плитки. Это значение представляет собой двойную ширину полей самой маленькой плитки:

![полная плитка без заполнения с полями](images/assetguidance14.png)

В этом примере поля слишком узкие:

![полная плитка без заполнения со слишком узкими полями](images/assetguidance15.png)
