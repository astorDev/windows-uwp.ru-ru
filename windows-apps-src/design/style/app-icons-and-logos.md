:::row:::
    :::column:::
        * Заголовок окна приложения * список приложений в меню «Пуск» * диспетчер задач и задач * ваше приложение Плитка * экрана-заставки вашего приложения * в Microsoft Store :::column-end:::

## <a name="microsoft-store-listing-image"></a>Изображение списка Microsoft Store

«Как указать изображения для вхождения Мое приложение в хранилище Microsoft?»

По умолчанию мы используем некоторые изображения из пакетов в хранилище, как описано в таблице в верхней части этой страницы (и других [изображений, которые предоставляют во время процесса отправки](https://docs.microsoft.com/en-us/windows/uwp/publish/app-screenshots-and-images)). Тем не менее имеется возможность запретить использование изображения логотипа в пакетах вашего приложения при отображении списка клиентов на 10 Windows (включая Xbox) хранилище, а вместо этого хранилища используется только те изображения, которые можно загрузить. Это позволяет контролировать внешний вид приложения в различных дисплеев в хранилище. (Обратите внимание, что ваш продукт поддерживает более ранних версий ОС, заказчики могут оставаться изображений из пакетов, даже при использовании этого параметра.) Это можно сделать в разделе **хранилище логотипы** **со списком хранилища** шага процесса отправки.

![Указание хранилища логотипы во время процесса отправки приложения](images/app-icons/storelogodisplay.png)

При установке этого флажка появится новый раздел с названием **магазина отображения изображений** . Здесь можно загрузить 3 размеры изображения, вместо изображений эмблемы из пакетов ваше приложение будет использовать хранилище: 71 x 71, 300 x 300 и 150 x 150 пикселей. Размер 300 x 300 не требуется, хотя рекомендуется предоставление всех размеров 3.

Дополнительные сведения см. [Отображение только загружаются изображения логотипа в хранилище](/windows/uwp/publish/app-screenshots-and-images#display-only-uploaded-logo-images-in-the-store).

## <a name="generating-all-assets-at-once"></a>Создание всех средств за один раз

Первый элемент меню на вкладке **Визуальные ресурсы** **Всех средств Visual**именно то, что его названия: создает каждый ресурсами приложению нажатием кнопки.

![Создать все визуальные ресурсы в Visaul Studio](images/app-icons/all-visual-assets.png)

Все, что нужно делать, является отдельным изображением, питания и Visaul Studio будет создать небольшой фрагмент, Средняя мозаика, крупного фрагмента мозаики, широкий мозаики, крупного фрагмента мозаики, значок приложения, экрана-заставки и упаковки логотипов для каждой коэффициент масштабирования.

Для создания всех средств одновременно:
1. Нажмите кнопку рядом с полем **источника** **...** и выберите изображение, которое вы хотите использовать. При использовании растрового изображения, убедитесь в том, что это по крайней мере 400 400 пикселов, чтобы получить четкие результаты. Векторные изображения лучше всего; Visual Studio позволяет использовать AI (Adobe Illustrator) и PDF-файлы. 
2. (Необязательно). Настройте эти параметры в разделе « **Параметры экрана** »:

    А.  **Короткое имя**: укажите краткое имя для вашего приложения.

    Б.  **Показать имя**: укажите краткое имя отображения на средний, ширину или больших фрагментов. 

    в. **Мозаичного фона**: укажите шестнадцатеричное значение или имя цвета для фона мозаики. Например, `#464646`. Значение по умолчанию— `transparent`.

    г. **Spash фон экрана**: укажите имя шестнадцатеричное значение или цвет для фона экрана spash. 

3. Нажмите кнопку **Создать**. 

Visual Studio создает файлы изображений и добавляет их в проект. Если вы хотите изменить ресурсы, достаточно повторить процесс. 

Средства масштабирования значок выполните это соглашение об именовании файлов.

*имя файла*- масштаб - .png*Коэффициент масштабирования*

Например:

Square150x150Logo шкала 100.png, Square150x150Logo шкала 200.png, Square150x150Logo шкала 400.png

Обратите внимание, что Visual Studio не создает логотип значка по умолчанию. Это потому, что фирменный логотип уникален и, вероятно, не соответствующие значки приложений. Дополнительные сведения содержатся в разделе [значка уведомления для приложений в статье UWP](/windows/uwp/design/shell/tiles-and-notifications/badges). 


## <a name="more-about-app-icon-assets"></a>Дополнительные сведения о средств значок приложения
Visual Studio создаст всех средств значок приложения, необходимые для проекта, но если вы хотите изменить их, помогает понять, чем они отличаются от других ресурсов приложения. 

Ресурс значка приложения появится в масса мест: на панели задач Windows, в представлении задач, сочетание клавиш ALT + TAB и правом нижнем углу мозаики начала. Поскольку ресурс значка приложения появится в так много места, он имеет некоторые дополнительные изменения размера и plating параметры не имеют других средств: средства «целевой размер» и «unplated» активов. 

### <a name="target-size-app-icon-assets"></a>Размер целевого приложения значок средства
Помимо размеров коэффициент стандартной шкалы («Square44x44Logo.scale-400.png») также рекомендуется создавать ресурсы «целевой размер». Мы называем эти активы целевого размера, так как они предназначены для определенных размеров, например 16 точек, а не определенного масштаба, например 400. Размер целевого средства — для поверхностей, которые не используют масштабирования системы Полка:

* список переходов начального экрана (компьютеры)
* нижний угол плитки начального экрана (компьютеры)
* сочетания клавиш (компьютеры)
* панель управления (компьютеры)

### <a name="unplated-assets"></a>Unplated ресурсы
По умолчанию Windows использует по умолчанию на основе целевой ОС на цветной сигнальной. Если требуется, можно предоставить на основе целевого unplated ОС. «Unplated» означает, что ОС будет отображаться на прозрачном фоне. Имейте в виду, что эти ресурсы будут отображаться на различных цветов фона. 

![ресурсы без основы и с основой](images/assetguidance22.png)

* Панель задач и эскиз панели задач (настольные устройства)
* Список переходов панели задач
* Представление задач
* ALT+TAB

## <a name="customizing-asset-padding"></a>Настройка средства заполнения

По умолчанию генератор средств Visual Studio применяет рекомендуемые заполнения какой-либо образ. Если изображения уже содержат поля или изображения полного выпуска за обрез, расширяющие конец фрагмента, можно отключить эту функцию, снимите флажок **Применить рекомендуемые дополнения** . 

### <a name="tile-padding-recommendations"></a>Рекомендации заполнения сверху
Если необходимо предоставить свои собственные дополнения, вот наши рекомендации по кости. 

Существует 4 мозаики размеров: маленький (71 x 71), Средний (150 x 150), всего (310 x 150) и больших (310 x 310). 

Каждый ресурс плитки имеет такой же размер, что и плитка, на которой он размещен.

![Плитка, показывая полный выпуск за обрез](images/app-icons/tile-assets1.png)

Если вы не хотите на значок, чтобы расширить границы фрагмента, можно использовать прозрачные пикселы в вашей ОС для создания дополнения. 

![плитка и основа](images/assetguidance05.png)

Если плитка маленькая, то значок должен иметь такую ширину и высоту, чтобы его размер составлял не более 66% от размера плитки:

![соотношения размеров на маленькой плитке](images/assetguidance09.png)

Если плитка средняя, то ширина значка должна составлять 66%, а высота 50% от размера плитки. Это позволит избежать наложения элементов в фирменной панели:

![соотношения размеров на средней плитке](images/assetguidance10.png)

Если плитка широкая, то ширина значка должна составлять 66%, а высота 50% от размера плитки. Это позволит избежать наложения элементов в фирменной панели:

![соотношения размеров на широкой плитке](images/assetguidance11.png)

Для крупных плиток ширина значка должна составлять 66%, а высота— 50% от размера плитки.

![соотношения размеров на широкой плитке](images/assetguidance12.png)

Некоторые значки ориентированы горизонтально или вертикально, другие имеют более сложную форму, которая не позволяет расположить их на квадратном участке в нужном размере. Значки, которые выглядят выровненными по центру, могут быть больше с одной стороны. В этом случае часть значка может выступать за контур рекомендуемого участка размещения при условии, что визуально этот значок занимает такое же пространство, как и квадратный значок:

![три выровненных по центру значка](images/assetguidance13.png)

В случае если используются полные ресурсы без заполнения, необходимо учитывать элементы, пересекающие поля и края плиток. Поля должны занимать не менее 16% высоты или ширины плитки. Это значение представляет собой двойную ширину полей самой маленькой плитки:

![полная плитка без заполнения с полями](images/assetguidance14.png)

В этом примере поля слишком узкие:

![полная плитка без заполнения со слишком узкими полями](images/assetguidance15.png)
