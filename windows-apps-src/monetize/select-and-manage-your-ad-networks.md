# Выбор рекламных сетей и управление ими


\[ Обновлено для приложений UWP в Windows 10. Статьи о Windows 8.x см. в [архиве](http://go.microsoft.com/fwlink/p/?linkid=619132) \]

Перед использованием рекламного посредника вам понадобится настроить учетные записи для каждой рекламной сети, которая будет использоваться в ваших приложениях. Мы рекомендуем сделать это перед добавлением элемента управления рекламным посредником в проект Microsoft Visual Studio. Кроме того, мы рекомендуем настроить учетные записи для как можно большего количества рекламных сетей — это обеспечит максимальную гибкость при оптимизации работы с рекламным посредником.

## Поддерживаемые рекламные сети и типы проектов

В настоящий момент рекламный посредник поддерживается для следующих рекламных сетей и типов проектов.

|  Рекламная сеть    | Приложения универсальной платформы Windows (UWP) на C# или Visual Basic с использованием XAML | Приложения Windows 8.1 на C# или Visual Basic с использованием XAML | Приложения Windows Phone 8.1 на C# или Visual Basic с использованием XAML | Приложения Windows Phone 8.x Silverlight |               
|-------------------------------------------------------------------------|----------------------------------------------------|----------------------------------------------------------|-----------------------------------|---------------|
| [Microsoft](#microsoft)                         | **Поддерживается**                                      | **Поддерживается**                                            | **Поддерживается**                     | **Поддерживается** |
| [AdDuplex](#adduplex)                                                   | **Поддерживается**                                      | **Поддерживается**                                            | **Поддерживается**                     | **Поддерживается** |
| [Smaato](#smaato)                                                       | Не поддерживается                                      | **Поддерживается**                                            | **Поддерживается**                     | **Поддерживается** |
| [AdMob (Google)](#admob)                                                | Не поддерживается                                      | Не поддерживается                                            | Не поддерживается                     | **Поддерживается** |
| [Inneractive](#inneractive)                                             | Не поддерживается                                      | Не поддерживается                                            | Не поддерживается                     | **Поддерживается** |
| [Vserv VMAX](#vserv)                                                    | Не поддерживается                                      | Не поддерживается                                            | **Поддерживается**                     | Не поддерживается | 

Некоторые типы проектов, например C++ с XAML и JavaScript с HTML, в настоящее время не поддерживаются рекламным посредником. Для этих типов проектов можно отображать рекламу от Microsoft без рекламного посредника. Дополнительные сведения см. в разделах [AdControl в XAML и .NET](https://msdn.microsoft.com/library/mt313186.aspx) и [AdControl в HTML 5 и JavaScript](https://msdn.microsoft.com/library/mt313130.aspx).

## Особые параметры и сведения для каждой сети


Ниже указаны особые сведения для каждой рекламной сети, включая сведения о настройке учетной записи и выставлении приложения. Кроме того, ниже перечислены обязательные параметры, которые необходимо указать при [отправке приложения и настройке рекламного посредника](submit-your-app-and-configure-ad-mediation.md) (и/или в подключенных службах для [проверки реализации рекламного посредника](test-your-ad-mediation-implementation.md)). Дополнительные сведения об использовании определенной рекламной сети см. на веб-сайте.

Помимо обязательных параметров, каждая рекламная сеть также предоставляет дополнительные необязательные параметры, которые можно настроить с помощью кода в приложении. Примеры см. в подразделе [Необязательные параметры рекламной сети](#optionalparameters) далее. Полный список необязательных параметров см. в документации для каждой рекламной сети. Рекламный посредник игнорирует или переопределяет некоторые параметры, которые перечислены в следующих разделах. Например, параметры, связанные с текущим местоположением, обычно перезаписываются информацией о местоположении пользователя, определенной при помощи функции определения местоположения в приложении.

Обратите внимание на то, что когда вы [добавляете элемент управления рекламным посредником](add-and-use-the-ad-mediator-control.md) и указываете, какие рекламные сети использовать, Visual Studio пытается получить необходимые сборки для некоторых рекламных сетей программным способом. Если какую-либо сборку не удается получить автоматически, ее можно установить, используя указанные ниже ссылки для рекламных сетей.

## Необязательные параметры рекламной сети


Помимо обязательных параметров, каждая рекламная сеть также предоставляет дополнительные необязательные параметры, которые можно настроить с помощью кода в приложении. Полный список необязательных параметров см. в документации для каждой рекламной сети. Чтобы установить эти необязательные параметры в коде, используйте свойство **AdSdkOptionalParameters** объекта **AdMediatorControl**.

В следующем примере показано, как задать свойству [CountryOrRegion](https://msdn.microsoft.com/library/windows/apps/microsoft.advertising.winrt.ui.adcontrol.countryorregion.aspx) значение двухбуквенного кода страны или региона пользователя для Microsoft Advertising.

```CSharp
myAdMediatorControl.AdSdkOptionalParameters[AdSdkNames.MicrosoftAdvertising]["CountryOrRegion"] = "IN";
```

В следующем примере показано, как задать параметры **Width** и **Height** для Smaato.

```CSharp
myAdMediatorControl.AdSdkOptionalParameters[AdSdkNames.Smaato]["Width"] = 300;
myAdMediatorControl.AdSdkOptionalParameters[AdSdkNames.Smaato]["Height"] = 250;
```
