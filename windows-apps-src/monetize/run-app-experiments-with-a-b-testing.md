# <a name="run-app-experiments-with-ab-testing"></a>Выполнение экспериментов в приложении с использованием A/B-тестирования

Можно использовать центр партнеров для определения удаленных переменных, которые вы можете получить во время выполнения из своих приложений универсальной платформы Windows (UWP) и варианты из этих значений можно протестировать с пользователями для определения наиболее эффективных значения для автомобильного поведение нужного пользователя. Приложение может использовать удаленные переменные для настройки поведения приложения, например покупок из приложения, процедура входа, заголовки и размещение рекламы.

Целью A/B-теста должно стать выявление варианта значений удаленной переменной, который, скорее всего, обеспечит оптимальные коэффициенты конверсии (например, больше покупок в приложении) благодаря более удобной работе в приложении. После определения вариант, можно сразу же закончить эксперимент и предоставить этот вариант всей аудитории пользователей из центра партнеров, без необходимости повторной публикации приложения.

## <a name="create-and-run-an-ab-test"></a>Создание и выполнение A/B-теста

Чтобы создать и выполнить A/B-тест, выполните следующие действия.

1. [Создание проекта и определение удаленных переменных в центре партнеров](create-a-project-and-define-remote-variables-in-the-dev-center-dashboard.md). Этот проект содержит переменные и значения переменных по умолчанию для ваших экспериментов.  
2. [Кодирование приложения для экспериментов](code-your-experiment-in-your-app.md). Используйте API-Интерфейс в Microsoft Store Services SDK для получения значений удаленных переменных из проекта, который вы создали в центре партнеров, использовать эти данные для изменения поведения тестируемой функции и отправить событие просмотра и события конверсии в центр партнеров.
3. [Определение эксперимента в центре партнеров ](define-your-experiment-in-the-dev-center-dashboard.md). Создайте в своем проекте эксперимент, определяющий уникальные цели и варианты для вашего A/B-теста.
4. [Запускать эксперимент и управлять им в центре партнеров ониторинга](manage-your-experiment.md). Активация эксперимента и использовать центр партнеров для просмотра результатов эксперимента и завершить его.

Пошаговое руководство, которое описывает весь процесс, см. в разделе [Создание и запуск первого эксперимента с использованием A/B-тестирования](create-and-run-your-first-experiment-with-a-b-testing.md).

## <a name="requirements"></a>Требования

A / b-тестирования в центре партнеров поддерживается только для приложений UWP.

Прежде чем начинать эксперименты с A/B-тестированием, необходимо настроить компьютер, на котором ведется разработка.

* Следуйте инструкциям, приведенным [здесь](../get-started/get-set-up.md), для настройки компьютера разработчика для разработки UWP.
* [Установите пакет Microsoft Store Services SDK](microsoft-store-services-sdk.md#install-the-sdk). Помимо API для экспериментов этот пакет SDK также предоставляет API для других функций, таких как отображение рекламы и направление пользователей в Центр отзывов для сбора отзывов о приложении.

## <a name="best-practices"></a>Рекомендации

Для получения оптимальных результатов рекомендуем соблюдать следующие инструкции во время экспериментов с A/B-тестированием.

* Проведите эксперимент только с двумя вариантами со случайным распределением вариантов 50/50.
* Эксперимент должен длиться по крайней мере 2–4недели для сбора достаточного объема статистически значимых и полезных данных.

## <a name="related-terms"></a>Связанные условия

|  Термин  |  Определение  |
|--------|--------------|
| Project    |   Коллекция удаленных переменных со значениями по умолчанию, которые доступны вашему приложению с помощью набора инструментов Microsoft Store Services SDK. Проект также может содержать один или несколько экспериментов с общими удаленными переменными.  |
| Эксперимент    |   Набор параметров, определяющих A/B-тест, который получат ваши пользователи. Эксперименты определяются в области проекта, и каждый из них состоит из следующего:эксперимент <p></p><ul><li>*Событие просмотра*, указывающее, когда пользователь начинает просмотр варианта, являющегося частью эксперимента.</li><li>Одна или несколько целей с *событиями конверсии*, которые указывают, когда достигнута цель.</li><li>Один или несколько *вариантов*, определяющих данные переменных, которые используются в эксперименте. Вариант *control* использует значения переменных по умолчанию, которые определены в проекте для эксперимента. Помимо варианта control эксперименты, как правило, содержат как минимум один дополнительный вариант с переменными значениями, уникальными для этого эксперимента. </li></ul>          |
| Код проекта    |   Уникальный идентификатор, который связывает ваше приложение с проектом в вашей учетной записи центра партнеров. Этот код необходимо использовать для подключения с использованием A / b-тестирования службы в коде приложения, чтобы получать переменные данные и регистрировать события просмотра и преобразования в центре партнеров. Дополнительные сведения см. в разделе [Кодирование приложения для экспериментов](code-your-experiment-in-your-app.md).<p></p><p>Каждый проект и все эксперименты в этом проекте связаны только с одним кодом проекта. Коды проекта можно использовать для разграничения различных наборов экспериментов. Например, у вас может быть один набор экспериментов, который вы предоставляете тестировщикам в своей организации, и другой набор экспериментов, который предоставляется только внешним пользователям приложения.  Приложение может ссылаться на несколько кодов проекта, если в нем реализовано несколько экспериментов.</p>         |
| Вариант    |   Коллекция, состоящая из одной или нескольких переменных, которые вы тестируете в эксперименте. Каждый эксперимент должен содержать хотя бы одну переменную и два варианта (включая элемент управления). Эксперимент может включать до пяти вариантов.           |
| Переменная    |  Значение, которое ваше приложение использует для инициализации свойства или некоторых других значений в приложении. Во время эксперимента значение переменной меняется в зависимости от варианта. После завершения эксперимента переменной назначается значение из варианта, который предоставляете всем пользователям вашего приложения. Переменные могут иметь следующие типы: string, Boolean, double и integer.
| Событие просмотра    |  Это произвольная строка, которая представляет действия при начале просмотра пользователем варианта, являющегося частью вашего эксперимента. Обычно это имя события в коде. Код вашего приложения передаст эту строку события просмотра в центр партнеров, если пользователь начинает Просмотр варианта. Дополнительные сведения см. в разделе [Кодирование приложения для экспериментов](code-your-experiment-in-your-app.md).
| Событие преобразования    |  Произвольная строка, представляющая целевой объект для цели эксперимента. Обычно это имя события в коде. Код вашего приложения будет отправлять эту строку события преобразования в центр партнеров, когда пользователь достигает цели. Дополнительные сведения см. в разделе [Кодирование приложения для экспериментов](code-your-experiment-in-your-app.md).
